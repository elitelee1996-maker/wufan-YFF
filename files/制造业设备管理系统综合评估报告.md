## 2. Go/No-Go 信号灯建议

### 🟢 绿色 (Go)
*   **业务匹配度**：高。简道云的表单引擎与流程自动化完美契合维修工单、巡检计划等业务场景。
*   **经济可行性**：中高。在采用标准离线方案与异步 ERP 对接策略下，预估毛利可达 39%。
*   **技术可实现性**：中。IoT 监控需依赖外部网关聚合，ERP 对接需处理限流与事务补偿，但均有成熟实践。

### ⚠️ 前置条件 (Conditional)
1.  **IoT 预期管理**：必须在合同中明确数据为“分钟级聚合”，非毫秒级实时监控。
2.  **ERP 接口规范**：签约前必须获取客户 ERP 系统的完整 API 文档及测试环境权限。
3.  **离线方案边界**：优先使用简道云官方 App 的离线能力，不接受定制化原生壳开发（除非预算增加 50%）。

## 3. 分维度评估详情

### 3.1 需求分析与 WBS
*   **覆盖范围**：涵盖台账、巡检、维保、IoT、备件、看板六大核心模块。
*   **复杂度分布**：IoT 监控（5/5）与 ERP 对接（4/5）为最高复杂度项，占总开发工作量的 40%。

### 3.2 需求评估结论
*   **标准功能**：设备台账、维修工单、备件进销存、基础看板。
*   **二开功能**：IoT 边缘网关脚本、ERP 双向同步中间件、离线数据冲突辅助表。
*   **不可实现/高风险**：毫秒级波形分析、强一致性实时库存扣减。

### 3.3 方案设计与原型
*   **IoT 架构**：采用“传感器 -> 边缘网关(聚合) -> 简道云 API”路径，有效降低写入频率。
*   **离线逻辑**：基于官方 App 缓存机制，网络恢复后自动触发同步队列。
*   **原型演示**：[点击查看 HTML 交互原型](files/设备管理系统原型.html)

## 4. 风险矩阵与应对策略

| 风险类别 | 风险描述 | 发生概率 | 影响程度 | 应对策略 |
| :--- | :--- | :---: | :---: | :--- |
| **技术风险** | ERP 接口限流导致同步失败 | 中 | 高 | 引入指数退避重试机制；建立人工对账日志表。 |
| **业务风险** | 一线工人抵触离线操作复杂性 | 低 | 中 | 简化移动端 UI，提供“一键同步”大按钮。 |
| **交付风险** | IoT 传感器协议碎片化 | 高 | 中 | 要求客户提供标准 Modbus/OPC UA 网关，不直接对接底层硬件。 |

## 5. 关键假设与免责

1.  **置信度标注**：本报告中 IoT 模块评估置信度为“中”，因其高度依赖客户现场的网络环境与硬件选型。
2.  **数据一致性**：ERP 对接采用“最终一致性”模型，允许存在分钟级的数据延迟。
3.  **后续待确认**：需与客户 IT 部门确认 ERP 服务器的防火墙策略及白名单机制。
