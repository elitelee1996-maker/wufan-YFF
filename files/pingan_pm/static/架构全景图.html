02 核心设计：数据汇聚页 ↔ 原生详情页 双模式
模式 A · 数据汇聚页

二开定制页面

iframe嵌入简道云 · 做"面"的全局视角
🌐
全局聚合视图
WBS树、甘特图、资源日历——跨多条数据的聚合展示，简道云原生做不到
🧮
复杂计算可视化
CPM前推/后推法、关键路径高亮、浮动时间色带——计算在Bridge层完成
🖱️
批量交互操作
拖拽排序、批量添加依赖、甘特图拖拽调整——需要自定义交互逻辑
🔗
嵌入数据链接
每条数据旁放置链接按钮，点击后打开简道云原生详情页，进入模式B
⇄
数据链接跳转
模式 B · 原生详情页

简道云原生页面

单条记录的多维度信息 · 做"点"的深度交互
✏️
原生CRUD能力
表单字段编辑、数据校验、联动规则——简道云原生就有，不需要二开重写
🔄
流程审批闭环
提交审批、回退、转交、加签——流程引擎是简道云的核心能力
🤖
智能助手 + 按钮

🎨
自定义详情页
单条数据的多维度展示——关联WBS列表、资源占用时间线、历史变更记录
03 典型场景：一次完整操作的双模式切换

🎬 场景：项目经理发现资源冲突 → 处理 → 闭环

1
数据汇聚页
在资源日历中发现吊车A的时间冲突
资源日历（05）以周视图展示所有资源占用，红色脉冲标注冲突时段。项目经理看到吊车A在周三周四被两个任务同时占用。
页面: 05-资源日历
模式: A（数据汇聚）
数据源: resource_occupy表
2
数据汇聚页
点击冲突任务旁的"查看详情"链接
资源日历中每个占用条旁有一个🔗链接按钮，点击后在新窗口打开该任务在简道云中的原生详情页。
操作: 点击嵌入链接
URL: jdy.com/open/entry/{data_id}
跳转: 模式A → 模式B
3
简道云详情页
在简道云原生详情页中查看任务全貌
自定义详情页展示：任务基本信息、依赖关系列表、资源分配、CPM计算结果（ES/EF/LS/LF/TF）、历史变更记录。这些信息来自多张关联表的聚合。
页面: wbs_node 自定义详情页
模式: B（原生详情）
能力: 多维度展示 + 关联查询
4
简道云详情页
直接在简道云中修改资源分配
利用简道云原生的表单编辑能力，将"二次吊装"的资源从吊车A改为叉车C。无需二开前端重写CRUD逻辑。
操作: 修改资源字段
能力: 原生CRUD + 数据校验
联动: 字段联动自动更新
5
智能助手
智能助手自动触发：重新计算CPM + 生成预警
数据变更触发智能助手 → 调用后端插件/Webhook → Bridge层的CPM Engine重新计算 → 结果写回wbs_node表 → 自动生成预警记录。
触发: 数据更新事件
链路: 智能助手 → Webhook → Bridge
写回: wbs_node + warning表
6
Bridge层
Bridge层接收Webhook，执行CPM重算
CPM Engine读取最新的wbs_node + dependency数据，执行前推/后推法计算，将ES/EF/LS/LF/TF批量写回简道云。
输入: wbs_node + dependency
计算: forward_pass + backward_pass
输出: batch_update → JDY
7
数据汇聚页
回到资源日历，冲突已消失
刷新资源日历页面，吊车A的冲突标注已消除。预警看板（07）中新增一条"资源冲突已解决"的提示。闭环完成。
页面: 05-资源日历 + 07-预警看板
模式: A（数据汇聚）
状态: 冲突已解决 ✅

05 关键设计决策
🎯
什么时候用数据汇聚页？
• 需要跨多条数据的聚合展示（甘特图、日历）
• 需要复杂计算（CPM、冲突检测）
• 需要自定义交互（拖拽、连线）
• 简道云原生前端做不到的场景
📝
什么时候用简道云原生页？
• 单条记录的字段编辑（CRUD）
• 流程审批（提交/回退/转交）
• 智能助手触发的自动化
• 按钮触发Webhook调用
• 自定义详情页多维度展示
⚡
Bridge层做什么？
• FieldMapper翻译widget_id
• CPM Engine计算调度
• 数据聚合：多表JOIN查询
• Webhook接收：智能助手回调
• 不做CRUD——CRUD交给简道云原生
