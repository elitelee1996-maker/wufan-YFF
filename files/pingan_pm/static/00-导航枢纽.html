🔄 典型使用场景：从创建到监控

1
项目经理创建项目 → 选择模板 → 批量导入WBS
打开「项目计划编制」，从模板库选择"大件运输标准模板"，一键导入24个任务节点和预设依赖关系。
打开 04-项目计划编制 →
2
调整依赖关系 → 添加/修改前驱后继
进入「依赖关系编辑」，根据实际业务调整任务间依赖。系统实时DFS检测循环依赖，发现环路立即阻断并高亮。
打开 01-依赖关系编辑 →
3
执行CPM计算 → 检查约束冲突
进入「约束调度」，系统自动执行前推法/后推法。设置关键节点的MSO/MFO约束，查看冲突面板中TF<0的任务。
打开 02-约束调度 →
4
查看甘特图 → 确认排程合理性
切换到「甘特图」视图，检查基准vs推算的双层对比。关注关键路径（红色）和浮动时间（半透明区间）。
打开 03-甘特图 →
5
检查资源冲突 → 做出调度决策
打开「资源日历」，查看本周资源占用。发现吊车A在周三周四有冲突，选择"替换资源"将二次吊装分配给叉车C。
打开 05-资源日历 →
6
保存基线 → 项目执行中定期对比
排程确认后保存基线快照。执行过程中打开「基线对比」，查看哪些任务延期、哪些提前，偏差一目了然。
打开 06-基线对比 →
7
日常监控 → 处理预警 → 闭环跟踪
每天打开「预警看板」，处理🔴严重预警。点击预警卡片查看详情，创建行动项分配给责任人，跟踪直到闭环。
打开 07-预警看板 →
