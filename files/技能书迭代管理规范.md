# 技能书迭代管理规范

> **适用范围**: 擎析 Agent 所有技能书
> **生效日期**: 2026-06-13
> **维护责任人**: 擎析 Agent 主 Agent

---

## 一、触发条件

以下任一情况发生时，必须启动技能书扫描与更新流程：

| 触发类型 | 触发条件 | 优先级 |
|---------|---------|--------|
| **版本更新** | 简道云发布新版本（V10.x.x / V11.x.x 等） | P0 |
| **功能上线** | 简道云官方公告新增功能（如自定义详情页、AI 功能、集成应用等） | P0 |
| **插件上新** | 插件市场新增重要插件（ERP/签章/发票/硬件等高频场景） | P1 |
| **项目复盘** | 项目实施中发现技能书描述与实际能力不符 | P1 |
| **定期巡检** | 每季度末（3/6/9/12月）执行一次全量扫描 | P2 |

## 三、更新流程

### 3.1 标准流程（5 步）

```
触发事件
  │
  ▼
Step 1: 信息收集
  - 读取简道云更新日志/官方公告
  - 提取新增功能、新增 API、新增插件
  - 确认版本号和时间线
  │
  ▼
Step 2: 影响评估
  - 逐本技能书检查是否受影响
  - 标注影响级别：P0（必须更新）/ P1（建议更新）/ P2（可选更新）
  │
  ▼
Step 3: 执行更新
  - 使用 edit 工具逐处修改
  - 新增内容标注 🆕，修改内容标注 🔄
  - 添加更新日志条目
  │
  ▼
Step 4: 验证确认
  - read 更新后的技能书，确认内容完整
  - 检查关联技能之间的引用是否一致
  │
  ▼
Step 5: 生成报告
  - 输出《技能书更新对比报告》
  - 存档至 files/ 目录
```

### 3.2 快速流程（紧急更新）

当 P0 级功能上线且影响当前项目评估时，可简化为：

1. 直接 edit 受影响的核心技能书（jdy-architecture-patterns + jdy-custom-dev-capabilities）
2. 添加更新日志
3. 在项目评估确认卡片中标注"技能书已更新"

## 四、迭代记录格式

### 4.1 技能书内嵌日志

每本技能书末尾必须包含"更新日志"章节，格式如下：

```markdown
## 更新日志
- **YYYY-MM-DD**: [变更摘要]。数据来源：[信息来源]。
- **YYYY-MM-DD**: [变更摘要]。数据来源：[信息来源]。
```

**示例**：
```markdown
## 更新日志
- **2026-06-13**: 新增自定义详情页范式（C）、集成应用范式（D）、AI 增强范式（E）；更新混合架构边界（L0-L3）；新增 AI 能力匹配步骤。数据来源：Layer 2 扫描 V10.19.0→V11.2.0。
```

### 4.2 独立对比报告

每次更新后，生成独立的《技能书更新对比报告》，命名规则：

```
files/技能书更新对比报告_YYYY-MM-DD.md
```

报告结构：
1. 执行摘要（TL;DR）
2. 变更明细（按技能书分列，含更新前/后对比表）
3. 影响分析（对评估流程的影响、对二开边界的影响）
4. 使用建议（评估时必须执行的检查）
5. 附录（更新日志汇总）

## 五、版本管理

### 5.1 版本号规则

技能书版本号与简道云版本号对齐，格式：

```
技能书版本 = 简道云基准版本 + 迭代序号

示例：
- V11.2.0-1：基于简道云 V11.2.0 的第 1 次迭代
- V11.2.0-2：基于简道云 V11.2.0 的第 2 次迭代
```

### 5.2 版本标记

每本技能书的 YAML frontmatter 中应包含版本信息：

```yaml
---
name: jdy-architecture-patterns
description: 简道云应用架构设计范式、表关系规范及混合架构边界定义。
version: V11.2.0-1
last_updated: 2026-06-13
jdy_baseline: V10.19.0-V11.2.0
---
```

### 5.3 历史版本存档

重大更新（P0 级）后，将更新前的技能书备份至：

```
files/skill-archive/YYYY-MM-DD/技能书名.md
```

- [ ] 独立对比报告已生成并存档
