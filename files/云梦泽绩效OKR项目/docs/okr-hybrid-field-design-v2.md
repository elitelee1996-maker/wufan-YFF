# 云梦泽OKR混合模式 — 字段设计决策（v2.0）

**版本**: v2.0 | **日期**: 2026-06-02 | **元决策**: 混合模式（OKR复盘驱动学习，绩效参考但不直接换算）

---

## 一、元决策确认

OKR与绩效的关系：**混合模式**。

- OKR进度不直接换算为绩效得分
- OKR复盘数据（自评进度、对齐贡献、gap）作为绩效校准会议的重要输入
- 0.7达成不被惩罚，保留挑战性目标的空间
- 强制分布和绩效打分基于独立考核模板，OKR数据是参考维度之一

## 四、进度计算逻辑

### 4.1 计算链路

```
KR.当前值 / KR.目标值 = KR完成率
    ↓
KR.对上级贡献 = KR.贡献系数 × KR.完成率
    ↓
O.self_progress = Σ(KR.对上级贡献) / Σ(KR.贡献系数)
    ↓
叶子节点: O.alignment_progress = O.self_progress
中间节点: O.alignment_progress = Σ(子O.权重 × 子O.alignment_progress) / Σ(子O.权重)
    ↓
O.progress_gap = O.alignment_progress - O.self_progress
```

### 4.2 计算示例

**部门级O"北京大区销售额达成"（假设无自身KR）：**

```
子O₁: 张三"新签客户" 权重60%  alignment=60%
子O₂: 李四"客户留存" 权重40%  alignment=90%

O.alignment_progress = (60%×60% + 40%×90%) / 100% = 72%
O.self_progress = 72% (无自身KR，等于alignment)
O.progress_gap = 0% → 绿色（健康）
```

**部门级O"北京大区销售额达成"（假设有自身KR + 子O）：**

```
自身KR₁: 贡献系数0.5  完成率80%  → 对上级贡献=40%
自身KR₂: 贡献系数0.5  完成率60%  → 对上级贡献=30%

O.self_progress = (40%+30%) / 1.0 = 70%

子O₁: 权重60%  alignment=60%
子O₂: 权重40%  alignment=90%

O.alignment_progress = (60%×60% + 40%×90%) / 100% = 72%

O.progress_gap = 72% - 70% = 2% → 绿色（健康）
```

**部门级O（gap > 0.15 场景）：**

```
O.self_progress = 30%       ← 自身KR严重滞后
O.alignment_progress = 85%  ← 但下属超额完成撑高了

O.progress_gap = 55% → 红色预警（上级责任逃避）
```

### 4.3 Gap预警规则

| gap范围 | 预警级别 | 含义 | 触发动作 |
|---------|---------|------|---------|
| gap > 0.15 | 红色 | 上级责任逃避：自身KR滞后，被下属撑高 | 强制上级复盘自身KR |
| 0.05 < gap ≤ 0.15 | 黄色 | 下属超额贡献，需确认方向 | 提醒上级关注自身KR |
| -0.05 ≤ gap ≤ 0.05 | 绿色 | 方向一致，贡献匹配 | 无 |
| -0.15 ≤ gap < -0.05 | 蓝色 | 可能是健康的挑战性目标 | 提醒对齐校准对话 |
| gap < -0.15 | 橙色 | 对齐断裂：自己做得好但对上级贡献不足 | 强制对齐校准对话 |

## 五、数据工厂实现方案

### 5.1 数据流1：KR → O.self_progress

```
输入: KR表
  ↓
字段设置: 对上级贡献 = 贡献系数 × KR完成率（已在KR公式中完成）
  ↓
分组汇总（按"所属目标"分组）:
  · 贡献之和 = SUM(对上级贡献)
  · 系数之和 = SUM(贡献系数)
  ↓
字段设置: self_progress = 贡献之和 / 系数之和
  ↓
输出: 同步到目标表的 self_progress 字段
```

### 5.2 数据流2：子O → O.alignment_progress

```
输入: 目标表（全量）
  ↓
数据筛选: 上级目标 ≠ 空（排除根节点，根节点单独处理）
  ↓
分组汇总（按"上级目标"分组）:
  · 加权进度 = SUM(权重 × alignment_progress)
  · 权重和 = SUM(权重)
  ↓
字段设置: alignment_progress = 加权进度 / 权重和
  ↓
  ↓ (根节点单独处理)
  ↓
输出: 同步到目标表的 alignment_progress 字段
```

### 5.3 定时更新策略

- 更新频率：每小时（数据工厂定时更新）
- 增量策略：全量重算（数据量在500条O以内时性能无压力）
- 如果后续数据量增长：改为智能助手Pro触发式更新（KR修改时触发增量计算）

## 六、智能助手Pro：权重校验 + Gap预警

### 6.1 权重之和校验

```
触发: 目标表修改时（权重字段或上级目标变更）
  ↓
查询: 同一上级目标下所有子O的权重
  ↓
计算: 权重之和
  ↓
条件: 权重之和 ≠ 100%
  ↓
动作: 推送提醒给O负责人 → "子O权重之和为X%，请调整为100%"
```

### 6.2 Gap预警推送

```
触发: 定时触发（每周一9:00）
  ↓
查询: 目标表中 progress_gap 绝对值 > 0.15 的记录
  ↓
循环: 逐条推送提醒
  · gap > 0.15 → "您的O被下属撑高55%，自身KR达成仅30%，请复盘自身KR"
  · gap < -0.15 → "您的O对上级贡献不足，gap=-25%，请发起对齐校准对话"
```

## 七、三个自定义详情页的差异化设计

### 7.1 普通员工视图

- 主展示：**self_progress**（"我的达成"）
- 参考展示：alignment_progress（"我的贡献"）
- gap 展示：小字标注，仅当 > 0.15 或 < -0.15 时显示
- KR列表：展示精力分配优先级（1-5星），不显示贡献系数
- 可编辑：KR当前值、精力分配优先级

### 7.2 团队负责人视图

- 主展示：**双进度并列**（自评 | 对齐贡献）
- gap 展示：醒目位置，颜色预警
- 下级O列表：每人一行，展示 alignment_progress + gap
- KR列表：展示精力分配优先级 + 贡献系数（可编辑贡献系数）
- 可编辑：下级O的贡献系数、权重

### 7.3 HRBP视图

- 主展示：**全局gap热力图**（按部门/层级聚合）
- 全公司O列表：筛选 gap > 0.15 或 < -0.15
- 绩效校准面板：对比OKR自评 vs 绩效评分
- 可编辑：无（只读），可导出报表

## 八、与前版设计的关键变更对照

| 变更项 | v1.0 | v2.0 | 原因 |
|--------|------|------|------|
| 进度字段 | 1个（目标进度=AVG） | 2个（self + alignment） | 混合模式需要双重视角 |
| 进度计算 | 简单平均 | 加权平均（贡献系数） | 擎析评估确认P0缺陷 |
| KR权重 | 1个（权重） | 2个（精力分配+贡献系数） | 圆桌讨论发现语义冲突 |
| Gap指标 | 无 | progress_gap + 预警 | 圆桌讨论确认为核心诊断指标 |
| 实现方式 | 聚合公式 | 数据工厂 + 智能助手 | 聚合公式不支持加权，改用数据工厂 |
| 详情页 | 三角色切换（二开） | 三套自定义详情页（原生） | Frank确认用原生方案 |
