# OKR系统应用架构设计

## 1. 业务背景

企业OKR目标管理系统，支持公司级→部门级→团队级→个人级的树形层级分解与串联展示。简道云作为数据录入后端，前端二开实现目标地图可视化（卡片+连线）。

核心需求：一个上级O由多个下级O支撑，纯树形结构，无交叉对齐。

## 2. 表清单

| 表名 | 类型 | 说明 |
|------|------|------|
| 目标表 (Objective) | 业务主表 | 存储所有层级O，通过关联数据→本表实现自引用树形结构 |
| 关键结果表 (KR) | 业务明细表 | 存储每个O下的可量化KR，进度追踪与聚合计算 |

## 3. 表间关系

### 关系总览

| 关系 | 基数 | 实现方式 | 说明 |
|------|------|---------|------|
| 目标表 → 目标表 | 一对多 | 关联数据→本表（上级目标字段） | 自引用：上级O拥有多个下级O |
| 目标表 → KR表 | 一对多 | KR表建关联数据→目标表（所属目标字段） | 一个O拆出多个KR |

> 关联数据展示（自动反向，不建控件）：
> - 目标表详情页 → 下级目标标签页（所有上级目标=当前O的记录）
> - 目标表详情页 → 关键结果标签页（所有所属目标=当前O的KR）
> - KR表详情页 → 所属目标信息

## 5. 数据流

### 5.1 目标创建流程

```
① 管理层 → 在简道云目标表 → 创建公司级O（上级目标=空，目标层级=公司级）
② 系统自动 → 详情页出现"下级目标"关联标签页
③ 部门负责人 → 在公司O详情页点击"新增下级O" → 上级目标自动预填为公司O._id
④ 系统自动 → 部门O的上级目标=公司O._id，树形关系建立
⑤ 同理 → 在部门O详情页新增团队级O、个人级O
```

### 5.2 KR创建与进度更新流程

```
① O负责人 → 在O详情页点击"新增关键结果" → 所属目标自动预填为当前O._id
② 系统自动 → KR完成率公式计算（当前值/目标值×100）
③ 系统自动 → O的目标进度聚合公式重算（AVG所有KR完成率）
④ 前端 → GET目标表API → 拿到最新进度值 → 更新进度环
```

### 5.3 前端读取与渲染流程

```
① 前端 → GET /api/v2/app/{app_id}/entry/{objective_entry}/data → 拿到目标表全量数据（扁平列表）
② 前端 → buildTree(flatList) → 按"上级目标"字段建树（parentId=null→根节点，parentId=X→子节点）
③ 前端 → 按层级分列布局（公司级列→部门级列→团队级列→个人级列）
④ 前端 → SVG连线（每条O的上级目标._id → 父卡片）
⑤ 前端 → 渲染O卡片（字段映射见第6节）
⑥ 用户点击O卡片 → GET /api/v2/app/{app_id}/entry/{kr_entry}/data?filter=所属目标={O._id} → 弹出KR列表
```

## 7. 前端建树核心逻辑

```javascript
// 简道云API返回扁平列表，关联数据字段返回被关联记录的_id
function buildTree(flatList) {
  const map = {};   // _id → node
  const roots = []; // 根节点数组

  flatList.forEach(item => {
    map[item._id] = { ...item, children: [] };
  });

  flatList.forEach(item => {
    const node = map[item._id];
    const parentId = item["上级目标"]; // null或上级O的_id
    if (!parentId) {
      roots.push(node);           // 根节点(公司级O)
    } else {
      const parent = map[parentId];
      if (parent) parent.children.push(node);
    }
  });

  return roots; // 通常只有1个根节点
}
```

## 9. 前端布局算法

采用**层级分列布局**（不是力导向图）：

- 第1列：公司级O（根节点，parentId=null）
- 第2列：部门级O（parentId指向第1列）
- 第3列：团队级O（parentId指向第2列）
- 第4列：个人级O（parentId指向第3列）
- 同列内按"所属组织"分组纵向排列
- 列间用SVG实线连线，从父卡片右侧连到子卡片左侧

## 11. 简道云搭建顺序

```
① 创建目标表 → 先建基础字段（编号、层级、组织、负责人、描述、类型、状态、周期、权重、备注）
② 创建KR表 → 基础字段 + "所属目标"关联数据→目标表
③ KR表 → 建"KR完成率"公式字段（当前值/目标值×100）
④ 回到目标表 → 建"上级目标"关联数据→本表（自引用）
⑤ 目标表 → 建"关联数据"字段指向KR表（聚合公式前提）
⑥ 目标表 → 建"目标进度"聚合公式 = AVG(KR完成率)
⑦ 目标表 → 建"关联数据"字段指向本表（反向，下级目标）
⑧ 目标表 → 建"下级目标数"聚合公式 = COUNT(下级目标)
⑨ 配置数据联动：选"上级目标"后自动带出其层级、组织等
⑩ 配置数据联动：KR表选"所属目标"后自动带出负责人
⑪ 手动创建1条公司级O作为种子数据（上级目标=空）
```

## 12. 自动化规划

| 规则名称 | 触发条件 | 执行动作 | 自动化级别 | 所属阶段 |
|---------|---------|---------|-----------|---------|
| KR进度更新通知 | KR表当前值被修改 | 推送提醒给O负责人 | L1(智能助手) | 二期 |
| O状态自动流转 | 所有KR完成率≥100% | O状态自动改为"已完成" | L1(智能助手) | 二期 |
| 周期切换提醒 | 新季度开始 | 推送提醒创建新季度O | L1(智能助手) | 二期 |

## 13. 分期落地计划

| 阶段 | 内容 | 目标 |
|------|------|------|
| 一期 | 目标表+KR表+关联数据+聚合公式(L0) + 前端基础树形展示 | 跑通数据录入和树形展示 |
| 二期 | 智能助手自动化(L1) + 前端交互完善(新增/编辑/筛选) | 自动化进度更新和通知 |
| 三期 | 仪表盘目标地图(自定义组件) + AI辅助拆解KR(L2) | 接近截图的卡片连线视觉效果 |

## 14. 搭建注意事项

1. **自引用字段创建时机**：目标表内必须已有至少1条记录，"上级目标"关联数据才能正常选择。先手动创建1条公司级O作为种子数据
2. **聚合公式依赖链**：目标进度依赖KR完成率，必须按顺序建：KR字段→KR公式→目标表关联数据→目标表聚合公式
3. **关联数据不支持多选指向本表**：简道云关联数据字段是单选的，每次只能选一条被关联记录。树形结构靠单选自引用实现，交叉对齐需用其他方式（如多行文本存编号）
4. **关联本表不允许自关联**：一条记录不能选择自己作为上级目标，简道云会自动排除当前记录
5. **流水号不可被智能助手赋值**：目标编号和KR编号用流水号，后续智能助手自动化时不能自动填写
6. **API版本要求**：简道云开放API需要企业版及以上，免费版和标准版无法使用API读写数据
