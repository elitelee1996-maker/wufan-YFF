## 一、顶层设计：全面绩效模型 (Total Performance Model)

### 1.1 核心理念

北森绩效管理的核心理念是"**全面支撑战略**"，通过五个维度构建管理闭环：

| 维度 | 说明 | 简道云映射 |
|------|------|-----------|
| **战略对齐** | 从组织目标到个人目标的层层解码 | 组织目标 → 员工目标 → OKR 三级联动 |
| **多元模式** | KPI/OKR/BSC/PBC 并存 | 多模板引擎 + 考核项配置 |
| **全过程管理** | 目标设定-过程辅导-考核评价-结果应用 | 表单+流程+仪表盘全链路 |
| **多源评估** | 360度评价（上级/同级/下级/外部） | 多人评估节点 + 匿名互评 |
| **全方位应用** | 绩效结果挂钩薪酬/晋升/发展 | 数据工厂 + API 集成 |

### 1.2 模块划分原则

按 **"员工自助" vs "管理配置"** 两大维度组织：

```
┌─────────────────────────────────────────────┐
│           北森绩效云功能全景                  │
├──────────────────┬──────────────────────────┤
│   员工自助模块    │     管理配置模块          │
├──────────────────┼──────────────────────────┤
│ • OKR            │ • OKR 管理               │
│ • 目标           │ • 员工目标管理            │
│ • 绩效           │ • 员工绩效管理            │
│ • 改进计划       │ • 组织目标管理            │
│ • 申诉           │ • 组织绩效管理            │
│                  │ • 战略解码                │
│                  │ • 数据分析                │
└──────────────────┴──────────────────────────┘
```

## 二、简道云复刻架构总览

### 2.1 技术架构：混合模式

采用 **"简道云零代码层 + 二开服务层"** 混合架构：

| 层级 | 简道云原生能力 | 二开服务补充 |
|------|---------------|-------------|
| **数据层** | 表单（普通/流程）、子表单、关联字段 | 复杂计算逻辑、跨系统同步 |
| **流程层** | 审批流、填写节点、插件节点 | 动态负责人、会签加签、自动提醒 |
| **展示层** | 仪表盘、聚合表、数据工厂 | 自定义 UI、Mermaid 图表 |
| **自动化** | 智能助手Pro（触发+执行） | HTTP 触发器、循环容器、API 调用 |
| **集成层** | 企业互联、第三方集成 | Webhook、自建插件、SSO |

### 2.2 核心功能模块映射

| 北森模块 | 简道云实现方案 | 关键功能点 |
|---------|---------------|-----------|
| **OKR** | 表单+流程+智能助手 | OKR制定/审批/协同/地图/四象限复盘 |
| **目标** | 表单+流程+数据工厂 | 目标制定/审批/协同/看板/电子签署 |
| **绩效** | 流程表单+多人评估+校准 | 指标制定/360评估/强制分布/结果确认 |
| **组织目标** | 组织维度表单+权限组 | 组织目标计划/代理人/定量分配 |
| **组织绩效** | 组织绩效活动+结果模块 | 组织评价角色/菜单授权 |
| **战略解码** | 战略解码计划+进度管理 | 战略与目标联动/审批流 |
| **数据分析** | 预置数据集+报表设计 | 6大目标数据集+4大绩效数据集 |

## 三、员工自助模块设计

### 3.1 OKR 模块

#### 功能清单

| 子模块 | 功能点 | 简道云实现 |
|--------|--------|-----------|
| **我的OKR** | OKR制定、审批、协同、地图、四象限复盘 | 流程表单 + 智能助手Pro |
| **OKR看板** | 制定分析(经理)、执行分析(经理)、结果分析(经理) | 仪表盘 + 聚合表 |
| **团队OKR** | 查看团队成员OKR、对齐关系可视化 | 关联子表 + 权限组 |

#### 关键字段设计

```
OKR主表:
- 周期ID（关联周期表）
- 员工ID（成员字段）
- Objective（文本，支持富文本）
- KR列表（子表单：KR描述/权重/完成值/信心指数）
- 对齐目标（关联字段，指向父级OKR）
- 可见范围（多选：全员/仅上级/指定人员）
- 状态（单选：草稿/审批中/已生效/已驳回）
- 电子签名（签名字段）
```

#### 流程设计

```
员工提交 → 直线经理审批 → （可选）虚线经理加签 → HRBP备案 → 生效
         ↓ 驳回
      员工修改后重新提交
```

#### 多人评估设计

北森支持 360 度评估，简道云实现方案：
- 使用 **流程表单的"多人评估"节点**
- 配置 **评价人规则**（上级/同级/下级/自评）
- 启用 **匿名互评** 选项
- 通过 **智能助手Pro** 自动计算加权得分

#### 强制分布方案

- 使用 **分布规则表** 定义各等级比例（如 A:20% B:70% C:10%）
- 通过 **数据工厂** 实时计算当前分布比例
- 在 **校准环节** 通过仪表盘预警超比例情况
- 支持 **图形化拖拽校准**（需二开前端页面）

## 六、关键技术实现要点

### 6.1 电子签署功能

**需求**：北森支持目标/绩效的电子签署

**简道云方案**：
1. 使用 **签名字段** 采集手写签名
2. 通过 **智能助手Pro** 触发二开服务生成 PDF
3. 二开服务调用 **打印模板 API** 生成带签名的 PDF
4. 将 PDF 回传至 **附件字段** 归档
5. 通过 **企业微信/钉钉** 推送签署完成通知

**限制**：免费版不支持自定义打印模板，需企业版及以上

### 6.2 多语言配置

**需求**：北森支持多语言（中英文切换）

**简道云方案**：
1. 为每个文本字段创建 **多语言版本字段**（如：objective_cn, objective_en）
2. 通过 **用户偏好表** 记录用户语言选择
3. 在 **仪表盘/表单** 中根据用户语言动态显示对应字段
4. 通过 **前端事件** 实现语言切换按钮

**限制**：需手动维护多语言字段，无自动翻译功能

### 6.3 IM集成（飞书/钉钉）

**需求**：目标任务集成IM，实时推送待办

**简道云方案**：
1. 启用 **第三方集成**（飞书/钉钉/企微）
2. 配置 **智能助手Pro** 的 HTTP 触发节点
3. 调用 IM 平台的 **消息推送 API**
4. 支持 **@提及** 功能（需在消息体中包含 @user_id）

**示例**：OKR 审批待办推送
```
触发条件：OKR状态变更为"审批中"
执行动作：HTTP POST 飞书机器人 webhook
消息内容：{
  "msg_type": "interactive",
  "card": {
    "title": "OKR审批待办",
    "content": "{{员工姓名}}提交了OKR，请及时审批",
    "actions": [{"type": "button", "text": "去审批", "url": "{{审批链接}}"}]
  }
}
```

### 6.4 强制分布与在线校准

**需求**：支持绩效等级强制分布（如 A:20% B:70% C:10%）

**简道云方案**：
1. **分布规则表**：定义各等级比例
2. **数据工厂**：实时计算当前分布比例
3. **仪表盘预警**：当某等级超比例时标红
4. **在线校准**（需二开）：
   - 开发自定义前端页面，支持拖拽员工头像调整等级
   - 调用简道云 API 批量更新绩效等级
   - 实时刷新分布比例图表

### 6.5 360度匿名互评

**需求**：支持匿名评价，保护评价人隐私

**简道云方案**：
1. 在 **多人评估节点** 启用"匿名评价"选项
2. 评价人看到的表单 **不显示被评价人姓名**（通过脱敏处理）
3. 评价结果汇总时 **隐藏评价人身份**
4. 通过 **权限组** 确保只有 HR 可查看原始评价记录
