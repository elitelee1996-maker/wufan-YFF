## 二、核心设计亮点

### 2.1 架构设计

采用 **"1+N+X" 文档架构**：
- **1个主文档**：精简版（≤3000字），提供整体框架和模块映射
- **N个工作表**：每个模块独立详细文档，便于维护和查阅
- **X个项目空间**：未来可为每个客户创建独立的项目实施文档

### 2.2 技术架构

采用 **"简道云零代码层 + 二开服务层"** 混合模式：
- **70%功能** 通过简道云原生能力实现（表单+流程+仪表盘+智能助手Pro+数据工厂）
- **30%复杂逻辑** 通过二开服务补充（PDF生成/矩阵可视化/API集成/多语言前端事件）

### 2.3 模块划分

按 **"员工自助 vs 管理配置"** 双维度组织：
- **员工自助**：OKR/目标/绩效/改进计划/申诉（面向全员用户）
- **管理配置**：OKR管理/员工目标管理/员工绩效管理/组织目标/组织绩效/战略解码/数据分析（面向HR和管理者）

### 2.4 数据模型

设计了 **完整的数据库 schema**：
- 核心表单：约 **50+ 张**（主表 + 子表 + 配置表）
- 聚合表：**10个预置数据集**
- 数据工厂：**8个数据流**（用于看板和分析）

### 2.5 流程设计

设计了 **完整的业务流程**：
- OKR审批流程（员工提交 → 经理审批 → HRBP备案 → 生效）
- 目标审批流程（员工提交 → 经理审批 → 电子签署 → 生效）
- 绩效考核流程（指标制定 → 自评 → 360评估 → 经理评分 → 校准 → 确认 → 电子签署）
- 战略解码流程（战略规划部提交 → BG负责人审批 → CEO审批 → 拆解为组织目标 → 拆解为员工目标）
- 绩效申诉流程（员工发起 → HRBP初审 → 重新评估/维持原结果 → 结案）

## 三、关键技术实现

### 3.1 电子签署功能

**实现方案**：
1. 简道云签名字段采集手写签名
2. 智能助手Pro触发二开服务API
3. 二开服务调用简道云打印API生成PDF
4. PDF回传至附件字段归档
5. IM推送签署完成通知

**限制**：需企业版及以上（支持自定义打印模板）

### 3.2 多语言配置

**实现方案**：
1. 为每个文本字段创建双语版本（如 objective_cn / objective_en）
2. 用户偏好表记录语言选择
3. 前端事件根据用户语言动态显示对应字段
4. 仪表盘为每种语言创建独立版本

**限制**：需手动维护多语言字段，无自动翻译

### 3.3 IM集成

**实现方案**：
1. 启用简道云第三方集成（飞书/钉钉）
2. 智能助手Pro HTTP节点调用IM机器人Webhook
3. 消息体包含交互式卡片和@提及功能

**支持场景**：审批待办/进度提醒/评估邀请/签署完成通知/申诉处理通知

### 3.4 强制分布与在线校准

**实现方案**：
1. 分布规则表定义各等级比例
2. 数据工厂实时计算当前分布比例
3. 仪表盘预警超比例情况
4. 在线校准需二开前端页面（拖拽调整等级）

### 3.5 360度匿名互评

**实现方案**：
1. 多人评估节点启用"匿名评价"选项
2. 评价人表单脱敏处理（不显示被评价人姓名）
3. 结果汇总隐藏评价人身份
4. 仅HR可查看原始评价记录
