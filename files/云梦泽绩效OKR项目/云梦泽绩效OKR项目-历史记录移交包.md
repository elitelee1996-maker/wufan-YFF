### 目标表核心字段

```
┌──────────────┬────────────────┬───────────────────────────────────┐
│     字段     │    控件类型     │              说明                  │
├──────────────┼────────────────┼───────────────────────────────────┤
│ 目标编号     │ 流水号          │ O-{YYYY}-{NNN}                    │
│ 上级目标     │ 关联数据→本表   │ 空=公司级根节点，自引用建树        │
│ 目标层级     │ 单选：公司级/部门级/团队级/个人级                   │
│ 所属组织     │ 部门单选        │ 组织树侧栏数据源                   │
│ 负责人       │ 成员单选        │ 卡片负责人                         │
│ 目标周期     │ 单选：年度/Q1~Q4                                    │
│ 目标描述     │ 多行文本        │ 卡片标题                           │
│ 目标类型     │ 单选：战略/运营/项目                                │
│ 目标状态     │ 单选：规划中/进行中/已完成/已暂停                    │
│ 权重         │ 数字            │ 在上级O中的占比                    │
│ 目标进度     │ 聚合公式        │ AVG(关联KR完成率)                  │
│ 下级目标数   │ 聚合公式        │ COUNT(下级目标)                    │
└──────────────┴────────────────┴───────────────────────────────────┘
```

### KR表核心字段

```
┌──────────────┬────────────────┬───────────────────────────────────┐
│     字段     │    控件类型     │              说明                  │
├──────────────┼────────────────┼───────────────────────────────────┤
│ KR编号       │ 流水号          │ KR-{YYYY}-{NNN}                   │
│ 所属目标     │ 关联数据→目标表 │ 指向父级O                          │
│ KR描述       │ 多行文本        │                                    │
│ 指标名称     │ 单行文本        │ 如"净回款额"                       │
│ 当前值       │ 数字            │                                    │
│ 目标值       │ 数字            │                                    │
│ 起始值       │ 数字            │ 基线值                             │
│ KR权重       │ 数字            │ 在所属O中的占比                    │
│ KR完成率     │ 公式            │ 当前值/目标值×100                  │
└──────────────┴────────────────┴───────────────────────────────────┘
```

### 表间关系

```
目标表 ──(上级目标: 关联数据→本表)──→ 目标表 (自引用建树)
目标表 ──(一对多)──→ KR表 (KR.所属目标 → O._id)
```

### 数据流

```
创建公司级O(上级目标=空) → 在详情页新增下级O(上级自动预填) → 逐层下钻
→ 在O详情页新增KR(所属目标自动预填) → KR完成率自动计算
→ O进度 = AVG(所有关联KR完成率) → 前端读取渲染
```

### 5.3 「权重」字段语义讨论（重要）

这是本次会话最关键的讨论，直接关系到表单设计。

**用户举例**：团队O 有5个维度（销售额35%、回款额25%...），个人O 也有同样5个维度。

**结论**：权重语义是统一的——**每条子目标对上级目标的贡献度**。

- 部门O 下挂团队O → 权重 = 这个团队对部门O完成的贡献占比
- 团队O 下挂个人O → 权重 = 这个人对团队O完成的贡献占比
- 个人O 下挂子维度 → 权重 = 这个维度对个人O完成的贡献占比（自然也是精力分配）

**三个要点**：
1. 权重不需要两层语义（"精力分配"和"贡献度"是同一枚硬币的两面）
2. 目标树从根到叶，每层权重加起来 = 100%，进度逐层加权向上汇聚
3. 部门O、团队O、个人O 都在同一张表里填，权重定义从上到下完全一致

## 六、OGSMA 战略解码框架（已沉淀）

### 五层角色分工

| 层 | 含义 | 参与执行？ | 说明 |
|----|------|-----------|------|
| O（目的） | 长期战略方向 | 否 | 一切的方向 |
| G（目标） | 可量化的阶段性结果 | 否 | G 是跨部门合力，不能挂在单一部门名下 |
| S（策略） | 达成 G 的具体路径 | 是 | **被部门O通过「战略来源」领走** |
| M（衡量） | 考核指标定义 | 否 | 注入考核模板 |
| A（责任） | 考核归属 | 否 | S.责任人是HR圈定考核组的锚点 |

### 关键设计决策

部门领的是 S（策略）而不是 G（目标），因为 G 天然跨部门（如"交易量突破1万亿"需要交易运营中心+金融服务事业部+产品技术中心合力达成），S 天然归属单一部门。
