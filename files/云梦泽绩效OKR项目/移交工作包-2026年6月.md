## 五、核心设计决策（必读）

### 5.1 绩效与OKR双轨独立

绩效管理和OKR管理是**完全独立的两条轨道**，各自有独立的计算逻辑、独立的页面入口、独立的数据流。暂时按没有关联设计。

| 维度 | 绩效轨道(KPI) | OKR轨道 |
|------|-------------|---------|
| 本质 | 底线指标，必须完成 | 挑战性目标，牵引成长 |
| 和薪酬关系 | 直接挂钩 | 不挂钩 |
| 目标设定 | 可达成的目标 | 挑战性目标，完成60-70%算成功 |
| 打分方式 | 完成率即得分 | 过程用完成率，期末可手动评分 |
| 上下级关系 | 拆解（自上而下分配） | 对齐（方向一致，各自独立） |

### 5.2 OKR对齐模式（飞书模式）

**O和O之间是「对齐」关系，不是「拆解」关系。**

- 下级O由下级自主制定，不是上级分配
- 对齐关系只表达方向一致，不触发任何计算
- 每个O的进度只由自己的KR决定，不从子O聚合

### 5.3 KR计算规则

```
KR完成率 = (当前值 - 起始值) / (目标值 - 起始值) × 100%（封顶100%）
O进度 = Σ(KR完成率 × KR权重) / Σ(KR权重)
```

### 5.4 下拉选项动态加载

前端下拉选项（目标类型、绩效周期）从简道云配置表动态读取，接口：

```
GET /api/okr/options
```

返回：
- `goal_types`: 从「目标类型设置」表读取
- `cycles`: 从「绩效周期」表读取（包含周期名称、类型、状态、起止时间）
- `statuses`: 固定值 ["草稿", "进行中", "已完成", "已关闭"]
