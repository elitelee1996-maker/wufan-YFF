# 简道云背景说明和介绍

- 简道云是一个零代码协作流程数据收集平台，因此前端可定制部分较少
- 简道云层级结构 ：应用 -> 表单|仪表盘|流程表单
- 可定制部分：api
  - 数据api：可以对某个表读写数据、上传下载表单文件附件等。
  - 应用和表结构api：只可以遍历所有app、只可以遍历所有entryid，查看表单内有那些字段（类型、名称、weightName 唯一名称、name 字段别名）
  - 流程api：提交、回退、转交、查询我的待办等一系列流程api
  - ……
- 前端可以定制的部分
  - 身份鉴权（重要）：可以把简道云的身份带到其他站点（通过OAuth2.0）。也可以带到iframe里边集成的站点。
  - 仪表盘嵌入iframe ： 可以嵌入固定链接
  - 数据详情页嵌入iframe ： 可以根据当前数据的某个字段拼接一个固定的出来。（注意可以联动Oauth2.0）
  - 弹窗：仅在单条数据记录编辑的模式下可用：点击弹窗按钮后，将会执行前端函数，可以接受表单的字段作为输入，然后启动弹窗到指定地址（模态框打开iframe），可在弹窗结束后返回信息到其他字段上。

- 限制部分：
  - 零代码搭建限制：简道云子表单最多200行
  - api速度受限：基本在10-20次/秒
  - api 可以读取到所有人的数据，但是没法以这个人的身份

- 后端插件：
  - py和js 环境执行的云函数：1分钟限制、有内存大小限制，沙盒环境，无法读取磁盘io，能利用的最多的就是
  - 主要应用场景：表单编辑时触发后端函数和前端函数调用后端函数、或者智能助手触发后端函数。
  - 智能助手：业务在记录提交更新、或者定时触发的一类。

## 触发流转说明
- 身份鉴权（重要）：可以把简道云的身份带到其他站点（通过OAuth2.0），也可以带到iframe里边。
1 用户提交表单记录 -> 智能助手 -> 智能助手触发后端插件 -> 触发到后端服务 -> 智能助手根据插件节点回写参数
2 用户提交表单记录 -> webhook推送 -> 服务接收推送
3 用户编辑记录时，点击按钮 -> 触发弹窗和弹窗iframe页面进行交互，可以传入一些字段，并返回一些字段信息。
4 用户打开应用 -> 打开仪表盘 -> 内嵌页面进行交互。
5 外部数据触发定制服务 -> 定制服务调用简道云api，写入简道云数据（可以触发简道云消息提醒） -> （可选）自动提交流程节点。
6 记录新建后设置唯一链接到记录上 -> 用户查看表单 -> 点击某条记录的链接 -> 导航到定制服务，根据参数查看记录对应数据
7 记录新建后设置唯一链接到记录上 -> 用户打开数据记录的数据详情页 ->  


## 开发模式：数据同步
一、数据同步类——简道云数据同步到第三方系统
  - 简道云提供了webhook，在数据删除、修改、新增、恢复的时候可以发送http webhook到定制服务器，定制服务把内容转为第三方系统可以接受的协议进行数据同步。
  - 简道云提供了api，可以采用定时轮询的方式去读取简道云数据。

二、数据同步类——第三方系统同步到简道云
  - 根据第三方系统的情况抓取数据并同步到简道云

举例场景：

企业内部自建系统	
1）消息通知，简道云待办
2）内部打卡核算。 
3）内部任务系统。
4）直连数据库。

使用的第三方信息管理平台	
1）仓库系统
2）商品库存系统
3）生产系统
4）人事系统
5）合同签章：法大大、上上签等
6）ai智能体
7）机器人插件：钉钉、飞书、企微机器人
8）其他综合平台：金蝶、用友等

其他场景系统	
1）短信平台
2）邮件平台。

## 开发模式：能力增强类
一、利用插件去增加表单能力：
   表现形式： 页面查询相关信息，并返回页面或者数据提交后，后台数据查询相关信息并返回到数据中。查询第三方系统数据，并返回简道云记录中。
    1、实时页面查询，通过简道云自定义插件可以在编辑时点击按钮，触发信息查询，典型的情况：文本框输入 公司名称，然后文本框失焦之后，触发调用后端函数 “工商信息查询”
   工作过程：点击左上角，导航到开放平台。导航到自建插件。然后新建（前端或者后端）插件，添加入参和出参，写好代码。返回到自建插件页面，启用这个插件。

## 开发模式：专有页面表现类
通过结合外部第三方页面，增强简道云表现能力。

一、通过嵌入到仪表盘，单独开一个页面。这个页面用来管理一众数据。（通过oauth2.0认证跟简道云做单点登录到定制页面服务）。

二、通过嵌入到数据详情页，查看数据时跟单独的数据进行交互。数据详情页可以挂 http://3rd-server.com/view?no=10031 这种， no 可以被作为参数传入。也可以传入别的参数，但是只有这一种传参方式且是固定的。

三、提交表单记录时搜索数据库数据并返回到子表单里。
    在新建和修改表单记录时，点击按钮，触发前端事件调用前端函数，弹出弹窗页面。然后云端页面展示数据库搜索列表，勾选需要返回的数据。关闭弹窗后，数据返回到简道云子表单上。

举例1：开发专有数据库搜索页面，增强简道云表现力。
举例2：共享文档编辑。简道云没有协作编辑文档的能力。客户找了第三方平台买了一个共享文档编辑的平台。需要上传文件时，不使用简道云自带的弹窗。先通过弹窗页面，让用户上传到对象存储中，并在定制服务连接的数据库中记录文件和id的关系。返回一个带有id的链接（比如 http://3rd-server.com/view?no=10031 ）到简道云记录中，这样后面的人打开这个链接就能参与编辑。（推荐，编辑时可以弹窗+链接。查看时走链接，或者数据详情页的iframe嵌入）
举例3：开发了一个看板，显示当前园区内人数。挂载在简道云仪表盘的“嵌入页面”组件上，这样就能有一定自定义页面能力了。
举例4：简道云自带的打印模板满足不了客户需求。就自行做了一个打印系统（可视化可拓展）搞了专门的页面可以拓展几张表的关联数据打印excel，用应用表单接口读取表的结构，通过api读取记录进行打印。每个模板可以有专门的触发打印的链接，放到简道云表单记录上，点击链接即可跳过去打印。当然花费人天也多。


## 开发模式：调用物理设备工作

通过读取和写入物理设备完成和物理设备交互。
一般有2个角色
1 边缘服务：用于连接到物理设备上然后读取或者操作物理设备：比如开发专门的服务来读取串口，从而读取地磅的实时重量，并上传到云端。
2 云端服务（或者纯内网的本地设备）：负责接收边缘服务的信息并下发指令到边缘服务用于和物理设备交互。（比如指示打印机打印）。同时肩负着和简道云表单交互的责任。比如提供页面给简道云查看，或者响应简道云推送的表。


举例1：读取地磅值：边缘服务通过串口读取实时重量上传到云端服务，在车辆入院登记简道云记录时，点击按钮，用后端函数请求云端服务的当前重量值，完成交互流程。
举例2：血压仪测量：用户在血压仪旁边，打开简道云表单。用户在简道云表单页面点击开始测量的按钮，弹窗函数打开云端页面服务，下发指令到边缘服务，边缘服务发生指令给血压仪。血压仪自动提示用户完成测量。测量后，弹窗自动关闭，并返回血压数值到页面字段上。点击提交后完成血压填报。



## 还有额外的章节
简道云集成到别的地方，有auth1完成这个对接，auth1利用简道云的基础能力，完成单点登录到简道云的能力。
比如钉钉单点登录到简道云
1 单点登录：可以从别的平台登录到简道云。
2 人员+部门同步：从钉钉读取所有的成员和部门同步到通讯录中。
3 消息推送：简道云里有待办和转交抄送之类的消息提醒，可以发给钉钉的应用消息或者通过定制服务发送到别的地方。
4 待办同步：同步简道云里边人员的待办到其他待办平台。

# 简道云 API 技能综述（详细版）

> 基于简道云开放平台官方文档浓缩整理
> 文档根地址：https://hc.jiandaoyun.com/open/10992
> 适用版本：v5/v6，企业版及以上（试用版可试用）

---

## 1. 通用调用规范

### 1.1 基础信息
| 项目 | 说明 |
|------|------|
| 统一地址 | `https://api.jiandaoyun.com/api` |
| 请求方式 | 统一 POST |
| 协议 | HTTPS |
| 编码 | UTF-8 |
| 数据格式 | JSON（文件上传为 form-data） |
| 全局频率 | 50 次/秒 |

### 1.2 鉴权方式
- 基于 API KEY 的 Bearer Token 鉴权
- HTTP Header: `Authorization: Bearer YOUR_APIKEY`
- API KEY 在「开放平台 >> 密钥管理 >> 创建 API KEY」生成
- 一个企业最多 500 个 API KEY

### 1.3 响应规范
| HTTP状态码 | 含义 |
|-----------|------|
| 2xx | 响应成功 |
| 400 | 接口错误（返回 `{code, msg}`） |
| 429 | 超出请求并发限制 |
| 502 | 网关异常 |
| 579 | 文件上传失败 |

### 1.4 核心 ID 体系
| 标识 | 含义 | 说明 |
|------|------|------|
| app_id | 应用 ID | 全局唯一 |
| entry_id | 表单 ID | app_id+entry_id 保证表单唯一性 |
| data_id / _id | 数据 ID | 数据全局唯一 |

## 2. 应用接口

> 文档：https://hc.jiandaoyun.com/open/12481

**⚠️ 仅支持查询，不支持通过API创建/修改/删除应用或表单**

| 接口 | 地址(V5) | 频率 | 说明 |
|------|----------|------|------|
| 用户应用查询 | `POST /v5/app/list` | — | 获取 API Key 授权范围内的应用信息 |
| 用户表单查询 | `POST /v5/app/entry/list` | — | 获取指定应用下所有表单信息 |

请求参数：
- `app_id` (String, 必需) - 应用ID（用户表单查询接口需要）

---

## 3. 表单接口（⚠️ 只读，不支持修改结构）

> 文档：https://hc.jiandaoyun.com/open/14216

**关键结论：简道云开放 API 不支持通过 API 修改表单结构。只能查询表单字段信息，无法通过 API 添加、删除或修改字段。**

表单结构的变更必须在简道云可视化界面中手动完成。

### 3.1 可用接口（仅查询）
| 版本 | 地址 | 频率 | 说明 |
|------|------|------|------|
| V5 | `POST /v5/app/entry/widget/list` | 30次/秒 | 推荐版本，app_id/entry_id放body |
| V2 | `POST /v2/app/{app_id}/entry/{entry_id}/widgets` | 5次/秒 | 路径参数方式 |
| V1 | `POST /v1/app/{app_id}/entry/{entry_id}/widgets` | 5次/秒 | 原始接口 |

### 3.2 请求参数（V5）
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| app_id | String | 是 | 应用ID |
| entry_id | String | 是 | 表单ID |

### 3.3 响应内容
| 参数 | 含义 |
|------|------|
| widgets | 字段信息列表 |
| widgets[].label | 字段标题 |
| widgets[].name | 字段名（有别名用别名，无别名用字段ID） |
| widgets[].widgetName | 字段 ID（_widget_ 前缀） |
| widgets[].type | 字段类型 |
| widgets[].items | 仅子表单有，包含子字段信息 |
| sysWidgets | 系统字段列表 |
| dataModifyTime | 表单数据最新修改时间 |

### 3.4 不支持 API 操作的字段
以下字段即使通过表单接口查询到，也无法通过数据 API 新建或修改：
- 分割线
- 手写签名
- 选择数据、查询
- 流水号（提交后系统自动生成）

## 4. 数据接口

> 文档：https://hc.jiandaoyun.com/open/14217

数据接口也叫数据 API，通过数据接口可以实现数据的增、删、改、查等操作。

### 4.1 接口总览

| 接口 | V5地址 | 频率 | 说明 |
|------|--------|------|------|
| 查询单条数据 | `POST /v5/app/entry/data/get` | 30次/秒 | 按 data_id 获取 |
| 查询多条数据 | `POST /v5/app/entry/data/list` | 30次/秒 | 按条件批量查询，按ID正序排列 |
| 新建单条数据 | `POST /v5/app/entry/data/create` | 20次/秒 | 向表单添加一条数据 |
| 新建多条数据 | `POST /v5/app/entry/data/create_batch` | 20次/秒 | 一次最多 100 条 |
| 修改单条数据 | `POST /v5/app/entry/data/update` | 20次/秒 | 按 data_id 更新 |
| 修改多条数据 | `POST /v5/app/entry/data/update_batch` | 20次/秒 | 一次最多 100 条 |
| 删除单条数据 | `POST /v5/app/entry/data/delete` | 20次/秒 | 按 data_id 删除 |
| 删除多条数据 | `POST /v5/app/entry/data/delete_batch` | 20次/秒 | 一次最多 100 条 |

### 4.2 查询单条数据接口

请求参数：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| app_id | String | 是 | 应用ID |
| entry_id | String | 是 | 表单ID |
| data_id | String | 是 | 数据ID |

响应：返回 `data` 字段，包含完整的单条数据（含系统字段和表单字段）。

### 4.3 查询多条数据接口

请求参数：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| app_id | String | 是 | 应用ID |
| entry_id | String | 是 | 表单ID |
| data_id | String | 否 | 数据ID（传入则只查这一条） |
| limit | Number | 否 | 返回数量，默认 10，最大 100 |
| filter | JSON | 否 | 过滤条件 |
| fields | Array | 否 | 指定返回字段 |

#### filter 过滤条件语法
```json
{
  "rel": "and",
  "cond": [
    {
      "field": "_widget_1529400746031",
      "method": "eq",
      "value": "简道云"
    },
    {
      "field": "createTime",
      "method": "range",
      "value": ["2022-01-01", null]
    }
  ]
}
```

| method | 说明 | 适用字段类型 |
|--------|------|-------------|
| eq | 等于 | 通用 |
| ne | 不等于 | 通用 |
| gt | 大于 | 数字、日期 |
| gte | 大于等于 | 数字、日期 |
| lt | 小于 | 数字、日期 |
| lte | 小于等于 | 数字、日期 |
| range | 范围 | 数字、日期、时间 |
| in | 包含任意一个 | 通用 |
| nin | 不包含任意一个 | 通用 |
| empty | 为空 | 通用 |
| not_empty | 不为空 | 通用 |
| like | 包含 | 文本 |
| not_like | 不包含 | 文本 |

### 4.4 新建单条数据接口

请求参数：
| 参数 | 类型 | 必需 | 说明 | 默认 |
|------|------|------|------|------|
| app_id | String | 是 | 应用ID | — |
| entry_id | String | 是 | 表单ID | — |
| data | JSON | 是 | 数据内容 | — |
| data_creator | String | 否 | 数据提交人（username） | 企业创建者 |
| is_start_workflow | Bool | 否 | 是否发起流程（仅流程表单） | false |
| is_start_trigger | Bool | 否 | 是否触发智能助手 | false |
| transaction_id | String | 否 | 事务ID（含附件/图片时需要） | — |

data 中的每个字段使用 `{ "value": xxx }` 格式：
```json
{
  "_widget_1432728651402": { "value": "简道云" },
  "_widget_1432728651403": { "value": 100 },
  "_widget_1432728651406": { "value": ["选项一", "选项二"] }
}
```

触发事件：新数据提交提醒、聚合表计算&校验、数据操作日志、数据量统计。不会触发重复值校验、必填校验。

### 4.5 修改单条数据接口

请求参数：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| app_id | String | 是 | 应用ID |
| entry_id | String | 是 | 表单ID |
| data_id | String | 是 | 数据ID |
| data | JSON | 是 | 数据内容（同新建） |
| is_start_trigger | Bool | 否 | 是否触发智能助手，默认 false |
| transaction_id | String | 否 | 事务ID |

子表单修改需注明子表单数据 ID（`_id` 字段）。

### 4.6 新建/修改多条数据接口

- 批量新建：`POST /v5/app/entry/data/create_batch`
- 批量修改：`POST /v5/app/entry/data/update_batch`
- 请求参数中的 `data` 为数组，每条数据格式同单条接口
- 一次最多 100 条

### 4.7 删除数据接口

单条删除：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| app_id | String | 是 | 应用ID |
| entry_id | String | 是 | 表单ID |
| data_id | String | 是 | 数据ID |

批量删除：
- `POST /v5/app/entry/data/delete_batch`
- 参数 `data_ids` 为数组，一次最多 100 条

## 5. 文件接口

> 文档：https://hc.jiandaoyun.com/open/13287

文件上传采用两步式：先获取上传凭证，再上传文件。

### 5.1 获取文件上传凭证和上传地址

| 项目 | 内容 |
|------|------|
| 地址 | `POST /v5/app/entry/file/get_upload_token` |
| 频率 | 20次/秒 |

请求参数：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| app_id | String | 是 | 应用ID |
| entry_id | String | 是 | 表单ID |
| transaction_id | String | 是 | 事务ID（推荐UUID，不含 `${var}` 等模式文本） |

响应：`token_and_url_list` - 每次最多返回 100 个凭证和上传地址。

### 5.2 文件上传

| 项目 | 内容 |
|------|------|
| 地址 | 上一步获取的 url |
| 方式 | POST form-data |
| 频率 | 20次/秒 |

请求参数（form-data）：
| 参数 | 必需 | 类型 | 说明 |
|------|------|------|------|
| token | 是 | String | 文件上传凭证 |
| file | 是 | 文件 | 要上传的文件（必须作为最后一个参数） |

响应：`key` - 文件key，用于在新建/修改数据时填入附件或图片控件的值。

### 5.3 注意事项
- token 有效期 1 小时
- transaction_id 有效期 1 小时，失效后 key 也无法使用
- 只有相同 transaction_id 的创建或修改请求才能使用该文件
- 图片/附件/手写签名/富文本中的 url 有效期为 15 天

### 7.2 部门实体（department）

| 属性 | 类型 | 含义 | 备注 |

| integrate_id | String | 集成模式同步关联ID | 仅集成模式返回 |
| seq | number | 部门排序 | v6新增，父部门内序号 |

### 7.3 成员实体（user）

| 属性 | 类型 | 含义 | 备注 |

| departments | Number[] | 所在部门编号列表 | — |
| type | Number | 成员类型 | 0=常规成员, 2=企业互联外部对接人 |

|------|------|------|

| name | String | 角色组名称 |
| type | Number | 0=常规角色组, 2=企业互联外部角色组 |
| status | Number | 1=使用中 |

### 7.6 版本演进

| 接口 | v1 | v2 | v4 | v5 |
|------|----|----|----|----|
| 成员 | _id作为id | username作为id | 新增type/status/integrate_id | 频率提升 |
| 部门 | _id作为id | dept_no作为id | 新增type/status/integrate_id | 频率提升 |
| 角色 | 无 | role_no/group_no | 新增integrate_id/type | 新增status，频率提升 |
| 企业互联 | 无 | 无 | username/dept_no + type/status | 频率提升 |

### 7.7 注意事项
1. 每个通讯录都是一棵部门树，根部门编号固定为 `1`
2. 公共模式与集成模式支持程度不同
3. 企业微信/钉钉的 integrate_id 与 dept_no 暂时一致；飞书部门ID为字符串，与 dept_no 不同
