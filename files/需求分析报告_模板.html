05 需求拆解与可行性评估

需求编号L1 需求模块L2 需求场景L3 需求描述实现方式产品 / 交付卡点建议方案风险等级需客户确认事项
SR-01项目立项与主数据CRM 合同触发项目建档CRM 收到合同后，自动或手动创建项目台账，带入客户、订单、合同交期、项目编号等基础字段，减少二次录入。轻量二开原因：CRM 是否开放接口、字段口径和唯一键尚未确认。示例/影响：若合同、订单、项目编号口径不一致，项目台账可能重复或漏建。方案1：CRM 暂不对接时，先通过项目立项流程和导入模板建档。方案2：CRM 提供接口后，通过 API / Webhook / HTTP 触发创建项目，并设置唯一键校验、失败待办和对账表。中CRM 接口文档、合同/订单唯一键、字段清单、是否允许先手工导入。
SR-02项目立项与主数据销售移交与补资料支持合同签订后销售/事业部移交资料，记录特殊条款、客户要求、交期变更和补资料状态。标准配置无配置销售移交流程表、资料清单普通表单、附件字段、节点字段权限和补资料提醒，运营管理部可追踪资料完整度。低销售移交清单模板、必须归档资料与仅留痕资料边界。
SR-03MPS 主计划主计划模板与 WBS 拆解按项目类型/产品类别维护 MPS 模板，支持主计划节点、责任部门、计划周期、前后置关系和预警规则。标准配置原因：不同产业和水处理项目颗粒度不一致。示例/影响：若模板过细，会增加维护成本；过粗则无法形成有效预警。配置项目类型、节点模板、节点责任矩阵和计划节点表；先以水处理产业试点模板落地，再抽象可复用模板。中一期模板覆盖哪些项目类型、节点是否固定、是否需要跨产业复用。
SR-04MPS 主计划编制、会签、发布与变更运营管理部编制主计划后，各部门限时确认，确认通过后发布；计划变更需发起审批、留痕并影响后续节点。标准配置原因：计划变更是否自动重算后续节点尚未明确。示例/影响：若变更后全部节点自动重排，需明确规则，否则容易造成验收争议。用流程表承载编制、会签、发布、变更；通过智能助手生成/更新节点任务，变更影响范围先按规则配置，复杂重排先做 Demo 验证。中变更审批路径、变更后是否重算后续节点、历史版本保留规则。
SR-05MPS 主计划节点任务自动拆解主计划发布后按节点责任部门/责任人生成待办任务，展示节点状态、计划时间、完成时间和超期状态。标准配置无配置计划节点明细表、任务表、关联数据、智能助手生成任务、状态回写和节点看板。低节点负责人来源、责任到部门还是到人、是否允许临时改派。
SR-06设计协同设备设计开始提醒按 MPS 提前 1 个月提醒水处理事业部/项目执行确认设备设计是否开始，并记录确认结果。标准配置无配置定时智能助手、待办通知、确认表单和逾期状态回写；支持移动端和电脑端提醒。低提醒提前量、提醒对象、未确认时是否升级到部门负责人。
SR-07设计协同设计资料清单与交付节点将设计文件、接口文件、技术协议、认可图等资料清单线上化，支持节点交付、负责人、版本状态和完成度统计。标准配置原因：设计资料数量与文件层级尚未定稿。示例/影响：若要求按每个物料号管理全部文件，会显著增加表结构和维护量。按“项目文件 + 物料类文件”先建普通表单，关联项目和物料类；用状态字段、负责人和交付节点控制完成度。中资料管理到项目、物料类还是物料号；是否一期只管关键资料。
SR-08设计协同图纸/认可图/意见单版本交互支持图纸、认可图、意见单多轮交互，记录版本、客户/设计院反馈、终版关闭和重新打开。轻量二开原因：客户提到单份文件可能出现大量版本，附件字段有单字段文件数、单文件大小和预览边界。示例/影响：若把全部版本大文件直接存入单字段附件，可能影响预览、归档和检索体验。方案1：标准表单记录版本台账，每轮意见单作为独立记录并上传附件。方案2：大文件或超多版本通过外部文件服务/对象存储承载，简道云保存链接、版本和审批状态。高单份文件最大版本数、单文件大小、是否必须在线预览、是否接受外部文件存储。
SR-09设计协同设计下生产与后下达物料异常监控设备/材料设计下生产时间，MPS 到期未下生产报警；设计报工后补下达物料作为异常或变更记录，用于复盘。轻量二开原因：物料类、物料号、采购单据层级需要定口径。示例/影响：从物料类下沉到物料号/订单行后，数据量和接口工作量会明显增加。一期按物料类或关键物料建立下生产节点；后下达物料通过变更/异常表记录。若需自动识别补下达，需对接 PLM/ERP/SAP 的报工和物料数据。高一期管理颗粒度、后下达判定规则、PLM/ERP 可提供字段。
SR-10采购协同采购订单签订监控按 MPS 到期监控采购订单是否按时签订，形成异常预警和责任追踪。轻量二开原因：采购订单数据来源可能在 SAP/SRM，是否自动同步未明确。示例/影响：若采购订单不维护或接口不可用，只能人工确认，自动预警准确性受限。标准配置采购节点确认表和逾期提醒；接口可用后通过 SAP/SRM API 同步订单号、签订日期、供应商、交期字段。中采购订单主系统、订单创建/签订字段、同步频率。
SR-11采购协同采购计划/订单/供应商交期数据接入汇总采购计划、采购订单、供应商交期、到货、检验等信息，为项目节点和看板提供数据。轻量二开原因：多系统接口和字段规则未形成责任矩阵。示例/影响：SAP/SRM/WMS 字段不一致会导致项目看板数据不可追溯。先形成接口交互清单；一期可用导入或人工维护补齐关键字段，接口阶段采用 API/中间表定时同步、失败重试和对账。高SAP/SRM/WMS 接口负责人、字段清单、项目编号/物料编码/供应商编码规则。
SR-12供应商与外购生产外购设备关键生产节点对外购设备按关键节点监控原材料齐套、开工、图纸确认、生产进度、报检、终检等大节点。轻量二开原因：客户明确不希望供应商登录多个系统，供应商节点宜优先沿用 SRM。示例/影响：若另建供应商入口，会增加供应商工作量并影响使用率。方案1：供应链管理部代维护关键节点。方案2：SRM 作为供应商入口，简道云同步关键节点。方案3：少量关键供应商通过企业互联/外部流程协同，但需确认账号边界。高供应商是否直接填报、SRM 是否已有字段、一期只管哪些关键设备/关键物料。
SR-13质量/ITP 管理外购 ITP 设备生产监控监控厂家 ITP 提交、事业部审核、采购催交、W/H 点资料、见证、会签和质量文件归档。轻量二开原因：W/H 点资料和质量文件附件量、会签规则未明确。示例/影响：大附件和多轮会签会影响表单设计和验收口径。标准配置 ITP 台账、W/H 点明细、报检/见证流程、会签流程、附件归档；若需从 SRM/质量系统自动获取文件，单独做接口。中ITP 模板、W/H 点清单、会签角色、质量文件是否来自供应商/SRM。
SR-14质量/ITP 管理自制 ITP 设备生产监控依据 MPS 触发编码、报审、传递装配制造中心，监控自制设备 W/H 点和 ITP 签字点。轻量二开原因：装配制造中心是否已有生产/报检系统及数据接口未明确。示例/影响：若仍线下维护，系统只能做到提醒和台账，不能自动判断完成。配置自制 ITP 流程、装配制造中心待办、W/H 点检查记录和会签；接口可用时同步报工/报检状态。中装配制造中心系统边界、报检记录字段、签字点是否需电子签名。
SR-15生产监控非 ITP 设备生产监控按生产日期和最终验收节点监控供应商备货、报检、终检，并进行到期预警。轻量二开原因：非 ITP 设备清单和供应商进度来源未明确。示例/影响：若依赖供应商人工反馈，数据及时性受使用率影响。配置非 ITP 设备台账、关键节点确认、逾期提醒；若 SRM 可提供节点，则通过接口同步。中非 ITP 设备范围、节点字段、责任方是供应链还是供应商。
SR-16生产监控自产排产与物料齐套根据车间产能排产，监控物料准备、报检、生产进度、过程检验点和齐套情况。轻量二开原因：排产属于车间生产管理能力，若要求复杂产能排程或 MES 级实时执行，不宜由简道云标准能力直接承诺。示例/影响：拖拽排产、强实时报工需外部系统或专门页面。一期用排产台账、齐套检查表、过程节点表和看板管理关键状态；复杂排程或 MES 数据通过接口同步或外部页面增强。高是否只做计划/状态监控，还是要求车间排产优化和实时生产执行。
SR-17生产监控生产异常处理闭环对生产异常点进行报检、跟踪、责任分派、处理进度推动和关闭留痕。标准配置无配置异常上报流程、责任分派、处理反馈、复核关闭、逾期提醒和异常统计仪表盘。低异常分类、责任归属、关闭标准和升级规则。
SR-18验收与交付预验收流程支持预验收触发、验收图纸确认、文件检查、视频/外观预验收、验收结论记录。标准配置原因：视频/大文件预览存在大小边界。示例/影响：若验收视频过大，需外部存储或只存链接。配置预验收流程、检查项、附件/链接字段、验收结论和权限；大视频按外部存储方案处理。中是否上传视频原文件、文件大小、预验收检查模板。
SR-19验收与交付问题整改闭环预验收问题反馈后，支持 +3 天确认整改方案、+7 天完成整改、整改进度实时监控和复验关闭。标准配置无配置问题整改流程、整改时限、责任部门、复验节点、逾期升级和整改台账。低整改时限是否固定、复验角色、整改延期是否需审批。
SR-20验收与交付包装与发运前置外购及自制设备按项目要求在规定周期内完成包装，并与发运前置条件关联。轻量二开原因：发运计划和包装记录的数据来源未明确。示例/影响：若 WMS/物流系统不对接，包装完成需人工确认。一期配置包装确认表、包装计划、完成记录和发运前置检查；WMS/物流接口作为扩展同步包装/发运状态。中包装周期、发运计划主系统、是否纳入一期发运管理。
SR-21变更与复盘计划/设计/物料变更管理对计划变更、设计升版、后下达物料、现场增补等事项进行发起、审批、留痕和复盘分析。标准配置原因：现场增补可能类似小合同，范围可大可小。示例/影响：若要求重新打开全部接口和流程，需作为独立变更项目评估。配置变更申请流程、变更类型、影响范围、审批、关联节点和复盘报表；大型现场增补按新项目/子项目处理。中变更类型、是否区分业主要求/设计错误/现场增补、是否影响合同范围。
SR-22提醒预警节点提醒、逾期和升级按 MPS 前置时间触发提醒，到期未完成报警，并可升级到责任部门负责人或运营管理部。标准配置原因：批量导入或接口同步数据不一定都触发智能助手。示例/影响：若节点状态来自批量同步，需用定时扫描方式补充预警。配置定时智能助手、状态字段、提醒策略、升级规则和提醒日志；接口/批量数据采用定时检查而非完全依赖提交触发。中提醒提前量、升级层级、通知渠道、是否需要企业微信/OA 待办。
SR-23看板与分析项目全局驾驶舱提供项目台账、进度台账、人员负载、延期预警、甘特图、采购/质量/生产分析和多项目下达及时率统计。轻量二开原因：看板指标依赖数据源和权限，跨系统数据实时性需确认。示例/影响：若指标来自 SAP/SRM/PLM 且无接口，看板只能展示平台内维护数据。标准仪表盘 + 数据工厂承载轻中量统计；复杂跨系统分析通过接口/中间表供数，权限按访问、取数、操作三层设计。中管理层指标清单、刷新频率、权限范围、是否需要下钻到原始数据。
SR-24权限与组织组织架构、角色和 SSO按运营管理部、水处理事业部、供应链、质量安全环保部、装配制造中心、信息化等角色控制数据范围和操作权限，必要时对接统一身份。轻量二开原因：复杂权限和 SSO 都依赖客户组织、账号来源和版本能力。示例/影响：一人多岗、跨项目授权和临时授权会增加权限矩阵复杂度。标准权限组、流程节点字段权限和仪表盘权限先落地；SSO/通讯录同步按客户身份平台接口单独配置或开发。中组织/岗位来源、一人多岗、项目级权限、是否必须 SSO。
SR-25附件与归档项目全生命周期资料归档归档项目过程中的合同、设计资料、质量文件、验收文件、包装资料、异常和变更记录。轻量二开原因：附件字段文件数、单文件大小、预览能力和批量迁移存在边界。示例/影响：大量设计文件或视频如果全部存平台内，可能带来容量和预览风险。标准附件表 + 资料分类 + 项目关联归档；大文件采用外部对象存储或文件系统，平台保存链接、版本和权限。高文件量、单文件大小、是否批量导入、是否必须在线预览/下载。
SR-26系统集成CRM/PLM/SAP/WMS/SRM/质量/OA 数据打通梳理并分阶段打通合同订单、设计资料、物料/BOM、采购订单、库存到货、供应商、质量/检验、OA 待办/组织权限等数据。深度二开原因：外部系统多、字段和接口责任未明确，且接口受频率、批量、鉴权和失败处理边界影响。示例/影响：没有接口文档和联调窗口时，不建议承诺确定工期。先输出接口责任矩阵；一期建议优先 CRM 项目建档、PLM 设计状态、SAP/SRM 采购关键字段，其余用导入补齐。接口采用中间表/API、队列、重试、对账和失败待办。高一期必须打通哪些系统、每个系统接口负责人、字段清单、联调窗口和失败补偿机制。
SR-27数据初始化模板导入与历史数据支持主计划模板、节点模板、项目台账、资料清单、关键设备/物料清单等基础数据导入；历史项目是否迁移待确认。轻量二开原因：历史数据质量、唯一键和关联顺序决定迁移工作量。示例/影响：在途流程和大量历史附件迁移会显著扩大范围。一期提供标准 Excel 导入模板和基础数据初始化；历史数据按客户清洗后的模板分批导入，复杂迁移另行评估。中是否迁移历史项目、迁移字段、附件是否迁移、客户是否负责清洗。
SR-28推广与运维水处理试点与多产业复制以水处理产业为试点沉淀流程、节点模板和看板口径，后续推广至船舶等其他产业。标准配置原因：不同产业节点标准化程度不同。示例/影响：水处理模板不能直接假设适配所有产业。一期固化水处理试点，输出可复制模板、字段口径和权限模型；推广前按产业差异做模板微调。中一期只上线水处理还是同步考虑其他产业；推广节奏和验收口径。

06 待二次确认风险项

编号风险项风险说明等级建议责任方确认节点
R-01接口范围不清CRM、PLM、SAP、WMS、SRM、质量/OA 多系统并存，当前仅明确方向，未形成接口字段和责任矩阵。高客户信息化部/各系统负责人方案确认前完成系统清单、字段清单、主数据归属和联调窗口。
R-02管理颗粒度下沉物料类、物料号、采购订单行、供应商生产节点层级未定，颗粒度越细，配置、接口和数据量越大。高运营管理部/水处理事业部/供应链Demo 阶段确定一期颗粒度，建议先管项目-物料类-关键设备大节点。
R-03供应商入口冲突客户明确不希望供应商登录多个系统；若另建供应商门户，使用率和维护成本风险高。高供应链管理部/信息化部优先复用 SRM 入口，简道云同步关键结果；供应商直填另行评估。
R-04设计资料版本和大附件设计文件版本可能很多，且可能包含大文件/视频，标准附件字段存在数量、大小和预览边界。高水处理事业部/信息化部明确文件量和存储策略，大文件采用外部文件服务或链接归档。
R-05看板指标数据来源项目驾驶舱依赖多系统数据，若外部数据不接入或维护不及时，指标准确性受影响。中运营管理部/信息化部逐项确认指标口径和数据来源，先上线平台内可闭环指标。
R-06计划变更重算规则MPS 变更后是否自动影响后续节点未明确，若规则复杂需 Demo 验证。中运营管理部/各业务部门确认变更审批、影响范围、重算规则和历史版本留痕。
R-07权限矩阵复杂跨部门、跨项目、一人多岗和领导看板权限可能复杂，不能只按角色名称判断。中信息化部/业务负责人上线前形成角色-数据-字段-操作权限矩阵。
R-08历史数据迁移如要求迁移历史项目、在途流程和附件，工作量与客户数据质量强相关。中客户业务/信息化部一期建议仅初始化模板和在手项目，历史迁移另行评估。
