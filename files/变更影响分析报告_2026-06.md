## 二、🔴 范式级变化（影响架构判定）

### 2.1 自定义详情页 — 全新架构范式

**变化轨迹**：

| 版本 | 更新内容 |
|------|---------|
| V10.10.0 | 🆕 **新增自定义详情页**（首次上线） |
| V10.12.0 | 支持移动端 |
| V10.13.0 | 支持组件显隐规则 |
| V10.14.0 | 新增列表组件 |
| V10.19.0 | 支持按钮/打印模板 |
| V11.0.0 | 「表单字段」组件优化、自定义工作台优化 |
| V11.1.0 | 开放更多数据源 |
| V11.2.0 | 新增图表组件 |

**影响分析**：

自定义详情页已经从"新功能"演变为一个**独立的架构范式**。它允许在简道云内部构建高度定制化的数据展示页面，支持图表、列表、简报卡片、按钮、打印模板等组件，且支持移动端和组件显隐规则。

**对技能书的影响**：

| 技能书 | 影响 | 建议动作 |
|--------|------|---------|
| `jdy-architecture-patterns` | 🔴 **重大** | 新增"自定义详情页范式"作为第 4 种架构范式，与表单/流程/仪表盘并列。L0-L2 层的能力边界需重新定义。 |
| `jdy-custom-dev-capabilities` | 🔴 **重大** | 前端可定制能力大幅增加。自定义详情页可能替代部分 iframe 嵌入场景，需更新"模式 C：专有页面"的决策树。 |
| `cost-estimation-model` | 🟡 中等 | 自定义详情页配置人天基准需新增（建议 2.0-4.0 人天/页，介于仪表盘和前端自定义页面之间）。 |

### 2.3 AI 能力矩阵 — 从零到体系化

**变化轨迹**：

| 版本 | AI 相关更新 |
|------|-----------|
| V10.16.0 | 🆕 **AI 实验室公测**：AI 助理、AI 推荐字段、AI 填充、AI 编辑助手、AI 执行节点、AI 生成公式（6 项一次性上线） |

| 版本 | AI 相关更新 |

| V10.3.0 | DeepSeek 插件 |
| V10.7.0 | AI 图像物品计数、AI 智能对话-豆包 |

| V10.11.0 | AI 实验室内测上线 |

**影响分析**：

简道云的 AI 能力已从"实验性"进入"体系化"阶段。AI 实验室的 6 大功能（助理、推荐字段、填充、编辑助手、执行节点、生成公式）覆盖了表单填写、数据处理、流程自动化等多个环节。

**对技能书的影响**：

| 技能书 | 影响 | 建议动作 |
|--------|------|---------|
| `jdy-architecture-patterns` | 🔴 **重大** | AI 能力应作为架构判定的新维度。某些原本需要二开的场景（如智能分类、自动填充）现在可能通过 AI 推荐字段/填充实现。 |
| `jdy-custom-dev-capabilities` | 🟡 中等 | AI 能力边界需明确：AI 推荐字段 vs 传统关联数据的适用场景区分。 |
| `ipd-standardization` | 🟡 中等 | IPD 需求拆解时需新增"AI 能力匹配"检查项。 |
| `cost-estimation-model` | 🟡 中等 | AI 功能配置的人天基准需新增。 |

## 三、🟡 能力级变化（影响二开边界）

### 3.1 智能助手 Pro 能力跃升

| 版本 | 更新 |
|------|------|
| V10.9.0 | 新增 HTTP 请求节点 |
| V10.19.0 | 新增「Agent」节点 |
| V10.20.0 | 部分节点支持多行文本输出 |
| V11.0.0 | AI 大模型节点支持输出子表单、节点级自动提交规则 |
| V10.4.0 | 新增发送消息节点 |
| V9.19.0 | 新增循环容器 |

**影响**：智能助手 Pro 的自动化能力大幅增强，HTTP 请求 + Agent 节点 + 循环容器 + AI 大模型节点，使得很多原本需要二开的自动化场景现在可以通过零代码实现。

| 技能书 | 建议动作 |
|--------|---------|
| `jdy-architecture-patterns` | L3 层"智能助手 + 少量二开"的边界需重新评估——很多场景可能不再需要二开 |
| `cost-estimation-model` | 自动化配置的人天基准可能需下调（原本需要二开的场景现在零代码可实现） |

## 六、基线修正

### ⚠️ 原基线文件存在严重错误

| 错误项 | 原基线值 | 实际值 |
|--------|---------|--------|
| 版本号体系 | V11.2.0 → V11.1.0 → V11.0.0 → V10.9.0 → V10.8.0 | V11.2.0 → V11.1.0 → V11.0.0 → **V10.20.0** → **V10.19.0** |
| doc_id 映射 | 25928, 25721, 25483, 25225, 25002 | 25928, **25185**, **25089**, **24920**, **23885** |
| 更新频率 | 每 2-3 周 | 约每 2 周（更频繁） |

**原因**：初始基线是基于主页面 markdown 模式抓取的侧边栏数据推断的，侧边栏中的 doc_id 指向的是帮助文档页面而非更新日志内容。text 模式抓取主页面才获得了正确的版本号和更新内容。
