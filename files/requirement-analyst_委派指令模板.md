# requirement-analyst 委派指令模板（🆕 v2.2 串行流程 + 产出容错）

## 执行时机

**阶段1**：主 Agent 完成阶段0 前置洞察后，先委派 requirement-analyst 拆解需求，产出 Sheet 1。

阶段2 的 knowledge-retriever 在阶段1 完成后才启动（用 Sheet 1 的节点列表做精准检索）。

## 委派指令结构

```
请对以下项目需求执行 IPD L1-L4 标准化拆解，输出「需求报价清单_[项目名].xlsx」的 Sheet 1：

## 项目信息
- 客户名称：[客户名]
- 项目名称：[项目名]
- 行业：[行业]
- 需求类型：[类型1: 客户明确文档 / 类型2: 售前调研清单 / 类型3: 碎片描述]

## 客户需求概要
[阶段0 提取的需求摘要和关键参数]

## 输出要求
1. 产出格式：**Excel (.xlsx)**，本次只产出 Sheet 1（业务需求清单）
2. 模板参考：先 `read_document` 读取「双瑞海洋项目运营管理平台_需求报价清单_最新规则.xlsx」
3. 层级规范：L1→L2 合并为「功能模块」列，L3 为「功能名称」列，L4 展开到「需求说明」列
4. 人天预估：加载 `cost-estimation-model` 技能，按基准填入「标准配置人天」和「接口/开发人天」
5. ⚠️ Sheet 3（风险清单）留空，由阶段2 经验检索反哺
6. ⚠️ Sheet 2（项目评估报价）和 Sheet 4（运维服务）留空，由主 Agent 后续合并填入
7. 模板说明：参考 `files/需求报价清单_模板说明.md`

## 关键行为准则
- 只输出 L1-L4，不涉及 L5
- 每个 L3 节点一行，L4 在需求说明列中展开
- 标注 AI 可实现 / 插件可实现 / 自定义详情页可实现（不计入二开）
- 标注权限需求（数据范围/操作权限/二开权限风险/身份透传）
- 人天标注「⚠️ 预估，待阶段2 验证」
- 报价理由列简要说明人天依据
```

## 🆕 v2.2 产出容错（2026-06-15 康尼机电复盘新增）

### 主产出
Excel (.xlsx) 文件，路径为 `需求报价清单_[项目名].xlsx`。

### 如果无法生成 xlsx（openpyxl 不可用 / 权限不足）
按以下 fallback 顺序尝试：
1. 生成 CSV 文件 `需求报价清单_[项目名].csv`（可用 pandas 或纯文本写入）
2. 生成 JSON 文件 `需求报价清单_[项目名].json`（结构化数据，主 Agent 可转换）
3. 在任务返回文本中输出完整的 Markdown 表格（需包含所有列，主 Agent 可解析）

### 关键
- **不允许只返回框架草案文本**。必须产出可被主 Agent 直接使用或转换的结构化数据。
- 如果连 JSON 都无法生成，在返回文本中明确说明原因（如"openpyxl 未安装，pandas 未安装，json 写入失败"），主 Agent 将自行生成。

## 委派后主 Agent 动作（🆕 v2.2 更新）

1. 子 Agent 完成后，**先验证产出物是否存在**：`glob(pattern="需求报价清单_*.xlsx")`
2. 如果 xlsx 不存在 → 检查返回文本中的结构化数据 → 主 Agent 自行用 `execute_python` + `openpyxl` 生成
3. 如果 xlsx 存在 → `read_document` 读取 Sheet 1，提取 L3 节点列表
4. 将节点列表作为输入，委派 knowledge-retriever（阶段2）
5. 阶段2 完成后，合并填入 Sheet 2/3/4，生成 HTML 报告
