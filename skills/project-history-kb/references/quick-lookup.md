## 按查询场景索引

### 场景1: 评估新项目工作量

**常见查询**:
- "制造业项目一般多少人天？"
- "教育行业项目报价多少合适？"
- "一个包含10个表单、5个流程的项目需要多少人天？"

**快速路径**:
1. 查阅 `industry-benchmarks.md` → 找到对应行业的人天和报价基准
2. 查阅 `requirement-patterns.md` → 根据功能类型估算工作量
3. 查阅 `pricing-benchmarks.md` → 参考7阶段报价基准

**示例回答**:
```
查询: "制造业项目一般多少人天？"
回答: 
- 人天中位数: 38天
- 人天均值: 50天
- 报价中位数: ¥85,000
- 成单率: 39%
- 建议: 制造业成单率较低，需精准控制报价
```

## 使用建议

### 1. 快速查询
- 使用本指南的"常见查询速查表"快速定位
- 根据查询类型选择对应的参考文件
- 直接查阅关键数据速记卡获取核心信息

### 2. 深度分析
- 根据场景选择对应的详细文件
- 结合多个维度进行综合分析
- 参考历史案例和应对建议

### 3. 决策支持
- 使用成单率分析评估项目价值
- 使用风险模式识别潜在问题
- 使用人员绩效数据选择团队

### 4. 持续优化
- 定期更新知识库数据（建议每季度）
- 收集新项目数据，验证历史基准
- 根据实际情况调整报价策略

## 数据时效性

- **数据范围**: 2022年8月 - 2025年5月
- **最后更新**: 2025年5月
- **建议**: 每季度更新一次知识库数据
