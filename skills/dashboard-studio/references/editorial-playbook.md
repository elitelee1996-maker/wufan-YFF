# Editorial Playbook · 数据期刊深度方法论

> 你正在做的是**数据期刊 / 深度报告 / 长卷叙事** — 不是仪表盘.
> 这份 playbook 锁住内容下限. 配合 `editorial-example.html` 食用.

---

## 一句话理念

**不要把数据 dump 成图表, 要从数据里榨出"被打动的瞬间"**.

- ❌ "本数据集 100 套样本, 房价跨度 23 万-210 万" (这是综述)
- ✅ "100 套房子里, 我们看到这座城市分成了三个圈层" (这是判断, 是被打动)

---

## ✨ 默认要超用户预期

除非用户**明确说**"简短 / 概要 / 几条要点", 否则按"顶级深度专业、内容量丰富"理解, 哪怕用户只说一句"做个分析报告":

- **超预期**: 把"做个报告"理解为"做个能截图发朋友圈、能拿去给老板汇报的深度作品"
- **内容规模**: **默认 ≥ 15 节** (数据撑得起就 ≥ 20 节, 不要克制)
- **多元表达**: 多角度观点 + 丰富 chart 形态 + 不同布局节奏, 不要"标题+段落+图"通篇重复
- **善用 Python**: 该多挖几轮就多挖几轮, **不要克制 Python 调用次数** — 数据没榨干就不要急着写 HTML

## 内容深度的 3 个硬指标

### ① 数据洞察 ≥ 15 个 (顶级 ≥ 25)

不只是数 KPI 算几个, 算法:
- 单维度统计 (均值/中位/分位/极端) — 1 个
- 多维交叉 (维度 A × 数值 B) — 1 个
- 排行 TOP / BOTTOM 双榜 — 1 个
- **反常识发现** (颠覆直觉) — 1 个 (权重 ×3)
- **业务场景洞察** (这数据对 XX 行业意味什么) — 1 个 (权重 ×2)

**< 15 个 = 浅薄, 回去用 `execute_python` 继续挖**.

### ② 反常识发现 ≥ 2 个 (顶级 ≥ 5)

直觉以为 X, 数据真相是 Y. 例:
- 直觉: "南向贵是采光好" → **真相: "南向贵是因为开发商把好朝向留给豪华小区"**
- 直觉: "100 套样本太少分析不出什么" → **真相: "100 套足以拆解出三层圈层结构"**
- 直觉: "Top 客户最重要" → **真相: "Top 客户 27%, Top 50 才 49%, 长尾占另一半"**

**没反常识 = 没"被打动的瞬间"**.

### ③ 金句数字 ≥ 3 个 (顶级 ≥ 5)

转化原则: **具象化 + 比较化 + 行业化**.

| ❌ 平淡 | ✅ 金句 |
|---|---|
| +139% 同比 | **2.39 倍** |
| 朝向 vs 单价 +53% | **南向溢价 199% — 比装修更值钱** |
| 1700 万 | **相当于全国每 80 人就有 1 人** |

## 📊 图表多样性硬要求 — ≥ 5 种, 不许 bar/pie 占全部

editorial 是数据期刊, 每个图表都是**视觉论证**. 一份 15 节报告 **必须用 ≥ 5 种不同 chart 形态**, 顶级 ≥ 7 种. **bar 跟 pie 加起来不许超过总图数的 50%**, 否则视觉单调到掉档.

### ❌ 反例 (Agent 最常犯)

```
12 张图全是: 8 个 bar + 3 个 pie + 1 个 line
→ 用户翻 3 节就疲劳, "图表都长一样"
→ 整份报告气质降一档, 像小学生作业
```

### ✅ 顶级输出长这样 (15-20 张图, 6-8 种形态)

| chart 类型 | 用什么数据 | 在 editorial 里典型场景 |
|---|---|---|
| **bar** (柱状) | 单维分类 × 数值 | 排行榜 / TOP N |
| **pie / donut** (环图) | 单维分类占比 | 整体结构 (≤ 5 类) |
| **scatter** (散点) | 两个连续变量 | 相关性 (面积 × 单价, 时长 × 转化) |
| **line / area** (折线/面积) | 时序数据 | 趋势变化 / 累计增长 |
| **radar** (雷达) | 多维评分 (≥ 4 维) | 多对象多指标对比 (3 圈层 × 6 维特征) |
| **stacked bar** (堆叠柱) | 单维 × 多类型构成 | 分组对比 (各季度 × 产品线占比) |
| **heatmap** (热力图) | 两维分类 × 数值 | 时间 × 类别密度 (周×时段订单热度) |
| **treemap** (矩形树图) | 多级嵌套占比 | 大类→子类层级 (国家→产品 GMV 占比) |
| **sankey** (桑基图) | 流量分流 | A → B 转化关系 (来源国 → 目的国) |
| **funnel** (漏斗) | 阶段流失 | 转化漏斗 / 客户分层金字塔 |
| **gauge** (仪表盘) | 单值 vs 阈值 | KPI 进度 / 健康度 |
| **boxplot** (箱线) | 分布异常 | 各组数据分位数对比 |

### 决策树: 你手上数据 → 该用哪种

```
你要呈现什么?
├── 比较多个项 (类别) 的大小 → bar / stacked bar (≥ 5 项用横向 bar)
├── 看占比构成 → pie / donut (类别 ≤ 5) 或 treemap (类别 > 5 或有层级)
├── 看时间趋势 → line / area / stacked area (多线对比)
├── 看两连续变量关系 → scatter (+回归线) / heatmap (密度)
├── 看多维度对比 → radar (≥ 4 维) / parallel (≥ 5 维)
├── 看流转/转化 → sankey / funnel
├── 看分布异常 → boxplot / histogram / violin
└── 看单值进度 → gauge / progress bar
```

**写完每个章节问自己**: 这个 chart 跟前一个章节是不是同款 (都是 bar / 都是 pie)? 是 → 换形态, 重新想数据该用什么图表表达更精准.

### 编码姿势 (ECharts)

每个 chart 都 spread 主题 base + 加 `safeRender` 包裹 (写一个错不要拖死其他):

```js
function safeRender(name, fn) {
  try { fn(); } catch (e) { console.warn('[chart]', name, e); }
}

safeRender('chart-scatter', () => {
  const el = document.getElementById('chart-scatter');
  if (!el) return;                              // ⚠ ID 对不上直接返, 不抛
  const ch = echarts.init(el);
  ch.setOption({
    ...bcEchartsBase,                            // ⚠ 必带主题 base
    series: [{ type: 'scatter', symbolSize: 12, data: [...] }]
  });
  new ResizeObserver(() => ch.resize()).observe(el);
});
```

**ID 自检**: write 完 HTML 后必须 grep 一下:
```bash
grep 'id="chart-' report.html       # HTML 里所有 chart 容器 ID
grep "echarts.init" report.html     # JS 里所有 init 调用
# 两份输出**完全对应**, 缺一不行
```

## 章节数下限 (按场景)

| 报告类型 | 章节下限 |
|---|---|
| 用户明说"简短" / Spotlight | ≥ 5 节 |
| 月报 | ≥ 8 节 |
| 季度复盘 | ≥ 12 节 |
| **默认 / 用户没明说要简短** | **≥ 15 节** |
| 年度回顾 / 深度专题 / 数据撑得起 | ≥ 20 节 (不要克制) |

**节数 < 下限 = 数据没挖透, 不是"精简"**. 没那么多内容, 是数据榨干没做够.

## 章节标题铁律

**章节标题是"判断", 不是"分类"**.

| ❌ 分类 (平庸) | ✅ 判断 (顶级) |
|---|---|
| "01 · 概况" | "01 · 三个圈层, 决定一切" |
| "02 · 朝向分析" | "02 · 南向溢价 199%: 朝向比装修更值钱" |
| "数据分析" | "第二增长曲线已浮现" |

**判断里通常含数字或动词**. 写好标题 = 半篇报告就成了.

---

## 节奏感: 避免视觉同质化

一份顶级 editorial, **N 节布局都不一样**. 参考 `editorial-example.html`, 它示范了 6 种布局:
- 整宽段落 + Hero 大字
- 6 列 KPI 阵 / 4 列 KPI 阵
- 1 大 + 2 小图表 (12 列 grid: 8 + 2×2)
- 整宽超大引语 (240px 引号 ghost)
- 黑底反差节 (timeline / 警报)
- Polaroid 拍立得卡 (案例展示)
- 触发器卡网格 (3-4 列)

**不要"标题 + 段落 + 图"通篇重复**.

## ❌ Editorial 不要做的事 (做了就是 dashboard)

- ❌ 加全局筛选条
- ❌ 加完整明细表 (排序+搜索+分页)
- ❌ 加下钻抽屉
- ❌ 用 KPI 阵 + 多 panel + 多 chart 的"驾驶舱"布局
- ❌ 章节标题写"概况" / "分析" 这种分类词

**Editorial 是作品, 不是工具. 被动消费, 不交互**.

## 4 步操作流程

### Step 1 · 摸排 (1-3 分钟)
```python
import pandas as pd
df = pd.read_excel('xxx')
print(df.shape, df.dtypes, df.head(10))
```

### Step 2 · 设计心智 (静态思考, 不操作)
进入 **Senior Editor** 姿态. 问 4 个问题:
1. 这数据里有什么**反常识**? (≥ 2 个)
2. 哪 3-5 个数字最有金句潜力?
3. 故事弧线 (起→承→转→合) 怎么走?
4. 我要分多少节? (按上面下限表)

### Step 3 · 榨干数据 (善用 execute_python, 该多挖就多挖)
不是 `df.describe()` 完事. 必须算:
- 多维交叉 (≥ 5 组)
- 排行 TOP + BOTTOM (≥ 2 套)
- 时间窗对比 (同环比 / 季节性, 如果有时间维度)
- **反常识专找** (≥ 3 个)

**目标**: 手里有 ≥ 25 个洞察候选 → 挑出 N 节最值得讲的. **挖不够就继续挖, 别克制 Python 调用次数**.

### Step 4 · 一次 write 整页 HTML
- 抄 `editorial-example.html` 的 `<style>` + 章节模板
- 替业务数据 + 文案
- 数据直接 inline `const D = {...}` (不要 data.json 外置)
- ECharts 必须 `...gpEchartsBase`
