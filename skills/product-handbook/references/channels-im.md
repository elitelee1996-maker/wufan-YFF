# 飞书 / 钉钉 / 企业微信集成

> IM 集成是平台最容易混淆的模块。同时存在 **CLI 接入套件**、**频道接入**、**自定义群推送** 三条独立路径，且各 IM 自己的限制不同。用户问"飞书怎么连"时先**问清他要做什么**再给方案。

## 三件不同的事

| 事 | 含义 | 解决什么 |
|---|---|---|
| **CLI 接入** (cli_managed 套件) | 把 IM 的 API 接到平台，作为**工具能力**给 Agent | "Agent 帮我发飞书消息 / 创建文档 / 查日历" |
| **频道接入** (channel binding) | 把 IM 群组绑到 Agent，用户在 IM 里 @bot 跟 Agent 对话 | "我想在飞书群里直接 @ Agent 跟它说话" |
| **自定义群推送** (Webhook) | 用 Webhook 工具调群机器人推消息 | "Agent 跑完任务后把结果推到企微某个群" |

**这三件事互相独立**:
- 装了 CLI 套件 ≠ 自动有频道
- 绑了频道 ≠ 自动能调 IM 工具
- 自定义群推送跟 CLI / 频道完全不相关

---

## 决策树: 用户说"我想让 Agent 跟飞书集成"

```
用户的核心需求是什么?
│
├─ "我想在飞书群里 @ Agent 跟它聊天"
│   → 频道接入。设置 → 频道，选飞书，按引导建机器人
│
├─ "我想让 Agent 在 Web 帮我处理飞书 (发消息 / 读文档 / 查日历)"
│   → CLI 接入套件。设置 → 能力套件，找飞书工作流，安装 → 扫码授权
│
├─ "我想让 Agent 在 Web 跑完任务后把结果推到飞书某个群"
│   → 自定义群推送。在飞书群加机器人 → 拷贝 Webhook → 平台配凭证 → 装推送工具
│
├─ "我想定时推日报 / 周报到飞书"
│   → 自动化 + (CLI 套件的发消息工具 / 自定义群推送 任选一)
│
└─ 用户也不知道想做什么
    → 先帮他理清: "你是想 Agent 在 Web 帮你处理飞书的事 (用 Web 跑)，还是想在飞书群里直接 @ 它聊天 (在飞书里聊)?"
```

## 路径 1: CLI 接入套件 (cli_managed)

**入口**: 设置 → 能力套件 → 找到 "飞书工作流" / "钉钉工作流" / "企业微信工作流" → 点"安装"。

### 安装走扫码授权

CLI 进程在沙盒里跑，输出二维码 URL，平台抓出来渲染成图给用户扫:

| IM | 扫码方式 |
|---|---|
| **飞书** (lark-cli) | OAuth device flow，飞书 app 扫一下即可 |
| **企业微信** (wecom-cli) | 扫码 + ASCII 二维码，企微 app 扫码授权 |
| **钉钉** (dws) | 扫码 + device code，钉钉 app 扫一下确认 |

授权后，平台收到 token 等凭据，自动写入连接凭证。此后任何用了对应 connection 的工具 / MCP 都能读到这些 token。

### CLI 接入完成后 Agent 能做什么

各 IM 套件自带一组工具，覆盖典型操作:

- 发送消息 (DM / 群)
- 查询通讯录、查日历、查 / 创建文档、查 / 创建表格
- 上传 / 下载云盘文件
- 提交日报 / 周报 (按模板)
- 查考勤 / 提交审批
- 创建 / 管理机器人

具体能力清单看各套件的 README。

### 常见限制 / 卡住点

**企业微信**:
- 企业 < 10 人才能用试用机器人 API。大企业要走"自建应用"路径，自己创建小应用拿 secret
- 企业管理员可能**禁用了对外 API** — 看到"该应用不允许外部调用"是后台关了，让用户找 IT 同学开
- 二维码授权后 token 默认有效期不长，过期要**重新扫码**

**飞书**:
- 团队管理员需要在飞书开放平台**审核应用**。用户授权后看到"等待审核"是飞书侧的事，平台无能为力，让用户找飞书管理员
- 部分接口要 **应用主管授权** (Tenant Access Token)

**钉钉**:
- 类似企微，需要钉钉管理员**给 Agent 应用开权限** (登录 / 通讯录 / 消息推送)
- 部分场景需要钉钉企业认证

## 路径 2: 频道接入 (Channel Binding)

让外部 IM 成为**对话入口** — 用户在 IM 群里 @ 一个机器人，像 Web 端一样跟 Agent 对话。

**入口**: Agent 配置页 → 频道 / 设置 → 频道。

### 绑定步骤

1. 选 Agent
2. 选 IM 平台 (飞书 / 钉钉 / 企微)
3. 平台引导用户去 IM 后台**自建应用 / 机器人**，拿到关键凭证 (callback URL / encrypt key / secret 等)
4. 在平台填回，通过验证
5. 把机器人**添加到目标群**，群里 @ 测试

### 频道与 CLI 的关系

**完全独立**。用户可以:
- 只装 CLI 套件 — Agent 在 Web 端可以发 IM 消息，但用户没法在群里 @ Agent
- 只接频道 — 用户在群里能 @ Agent，但 Agent 自己不会主动发 IM 消息 (除非另装 CLI 套件)
- 都装 — 双向打通

### 频道接入的限制

- **群聊 vs 单聊**: 默认只有单聊在 Web 端可见，群聊默认隐藏 (跨群可见性按管理员配置控制)
- **每个 Agent 一个机器人**: 一个 IM 应用绑一个 Agent。想让多个 Agent 都进 IM，要建多个机器人
- **管理员审核**: 跟 CLI 一样 — 飞书 / 钉钉 / 企微都要 IM 管理员开放应用接收外部消息的权限

## 路径 3: 自定义群推送 (Webhook)

最简单的"Agent → IM 群"方案 — 不用 CLI，不用频道，只要群里加个**群机器人**，把 Webhook URL 配成连接凭证 + 用一个调用工具即可。

### 步骤

1. **在 IM 群里加群机器人**
   - 飞书: 群设置 → 群机器人 → 添加 → 自定义 → 拷贝 Webhook URL
   - 企微: 群右上角 → 添加群机器人 → 拷贝 Webhook
   - 钉钉: 群设置 → 智能群助手 → 添加机器人 → 自定义 → 拷贝 Webhook

2. **平台配置连接凭证**
   - 设置 → 连接凭证 → "+ 新建连接凭证"
   - 字段: webhook_url (URL 类型，team 级)

3. **装一个推送工具**
   - 探索市场: 平台有内置的"飞书群推送" / "企微群推送" / "钉钉群推送"工具，搜索安装
   - 自创: 用 `tool-connect` 内置技能写一个，`requires_connections` 引用上一步的连接 slug

4. **装备给 Agent**
   - Agent 配置页 → 工具 → 装上对应推送工具

之后用户跟 Agent 说"把这个发到企微 XX 群"，Agent 就能调推送工具发了。

### 跟"自动化推送"的区别

平台还有**自动化** (Pipeline / 工作流) 功能 — 设置 → 自动化，可以配 cron / 触发器，让任务**定时自动跑**，跑完结果推到指定频道 (单聊 / 群聊)。

**配置流程**: 自动化面板 → 新建 Pipeline → 选 Agent → 配定时 / 触发条件 → 选输出频道 → 保存。

跟"对话里 Agent 调推送工具"的区别:
- **对话发**: 用户主动让 Agent 推 (一次性)
- **自动化**: 定时 / 事件驱动，不用人触发

## 用户卡住的常见症状

### "装了飞书 CLI 套件，Agent 调工具说没权限"

排查:
- 凭证是否真的扫码完成了 (设置 → 连接凭证 → 找飞书 → 看字段是否填齐)
- 用户是否在飞书侧把应用授权撤销了 (token 失效，重新扫码)
- 飞书管理员是否禁用了应用 (找 IT 同学)

### "接了频道，但群里 @ 机器人没反应"

排查:
- IM 后台机器人是否启用了"接收消息事件"
- callback URL / encrypt key 是否填对
- 平台侧是否正确收到回调 (设置 → 频道 → 找到目标频道 → 看消息日志)
- 群里机器人是否真的在群里 (刚拉进群可能要几秒生效)

### "企微 CLI 装不上"

八成是企业 ≥ 10 人但用了试用机器人 API。让用户走"自建应用"路径，先在企微的"自建小应用"创建一个真实应用拿 secret。

### "钉钉群里 @ Agent 没反应"

钉钉企业版要管理员先**认证企业** + **开启外部应用消息推送权限**。让用户找钉钉管理员配。

## 一行心法

CLI 接入 / 频道接入 / 自定义推送是**三件不同的事**。用户问"怎么连飞书"先**问清他要做什么**，别一上来推一条路 — 多半推错。
