# 核心概念字典

> 当用户问"什么是 X"或在使用 X 时表达困惑，给出**真实正确**的概念解释，不要凭通用平台经验编。

## 空间 (Space / Team)

**空间 = 团队 = 资源容器**。平台所有资源 (Agent / 资产 / 模型配置 / 凭证 / 文件 / 用量配额) 都挂在某个空间下。

| 类型 | 是什么 | 谁能进 | 适合 |
|---|---|---|---|
| **个人空间** | 注册时自动创建，名字默认"<你的名字> 的空间" | 仅本人 | 个人玩 / 自己用 Agent |
| **团队空间** | 协作空间，多人共享同一套 Agent / 资产 / 凭证 / 模型配置 | 所有者邀请的成员 | 团队协作 / 企业内部 |

**关键**: 切换空间 = 切换工作环境。在 A 空间建的 Agent / 资产，B 空间看不到。模型配置、连接凭证、订阅、配额都按空间隔离。

**入口**:
- 查看自己加入的所有空间: 设置 → 加入的空间
- 切换空间: 左下角空间切换器

**何时需要团队空间**: 多人协作 / 共享 Agent / 共享凭证 / 配额合并使用。

## 成员与角色

| 角色 | 能做什么 |
|---|---|
| **owner** (所有者) | 一切操作 (含删除空间、转让所有权、邀请/移除成员) |
| **admin** (管理员) | 大部分管理操作，但不能删空间 / 不能管 owner |
| **member** (成员) | 使用空间内资源，创建自己的 Agent / 资产 |
| **viewer** (访客) | 只读 |

**入口**: 设置 → 成员管理 (仅 owner / admin 看得到这一项)。

**邀请成员**: 设置 → 成员管理 → "邀请成员"按钮 → 输入邮箱 / 用户名 / 帆软通行证 ID。

**容量**: 团队默认上限 30 成员，订阅付费版可大幅扩容。配额已满时邀请会报"团队成员数量已达上限"。

## 智能体资产

"智能体资产"是平台的统一概念，指 **Agent 可以装备使用的能力单元**。5 种类型:

| 类型 | 是什么 | 例子 |
|---|---|---|
| **技能** (Skill) | SKILL.md + 配套资源 (脚本/模板/参考文档)。Agent 触发关键词时 read SKILL.md 获得专业知识或工作流 | 数据分析与可视化、前端设计、PPTX 生成 |
| **工具** (Tool) | 一段可执行 Python，提供 Agent 一个新动作能力 | read_file、bash、generate_image、自定义 API 工具 |
| **伙伴** (Partner / 子 Agent) | 一个独立的 Agent，被主 Agent 通过 `task` 工具调用执行子任务 | "代码审查专家"伙伴、"数据科学家"伙伴 |
| **MCP** | Model Context Protocol 服务器，第三方工具集协议，装上后获得一组工具 | 飞书 MCP、GitHub MCP、企查查 MCP |
| **卡片** (Widget) | UI 组件，Agent 在对话中渲染给用户做交互 | 询问卡、方案卡、问卷卡 |

**入口**: 设置 → 智能体资产 / 左侧栏 Agent 配置区。

**装备**: 资产装到空间里 ≠ Agent 能用。还要在 Agent 配置页**装备**给具体 Agent。每个资产对 Agent 有 4 态绑定:

| 态 | 含义 |
|---|---|
| **enabled** (启用) | Agent 一直能用 |
| **disabled** (禁用) | 装着但 Agent 看不到，用不了 |
| **optional** (按需) | Agent 默认看不到，用户在对话能力选择器里勾上才用 |
| **locked** (锁定) | 强制装上，用户不能取消 (典型: 平台默认带的核心工具) |

**Agent 不能直接装/卸资产、装/卸装备** — 这些都是用户/管理员的设置页操作。详见 `assets-management.md`。

## 能力套件 (Capability Pack)

预置的、即装即用的资产组合包。一个套件里可能包含工具 + 技能 + MCP + 凭证规范，整体描述一个完整能力 (如"飞书工作流"、"代码审查"、"数据分析师")。

**入口**: 设置 → 能力套件。

跟单个资产的区别: 套件**打包安装**，一次装一组；单个资产可以独立装。

**特殊类型 — CLI 套件 (cli_managed)**: 飞书 / 钉钉 / 企微三个 IM 接入是典型。安装时走扫码授权流程，详见 `channels-im.md`。

## 主 Agent vs 伙伴 (子 Agent)

**主 Agent**: 用户直接对话的 Agent。一个空间可以有多个主 Agent，在左侧栏 Agent 选择器切换，每个有自己的资产装备和系统提示词 (agents.md)。

**伙伴 (子 Agent)**: 不直接面对用户，由主 Agent 通过 `task` 工具调用。用于把一类专业子任务委托出去，独立的上下文 + 装备 + 系统提示词。

| 维度 | 主 Agent | 伙伴 |
|---|---|---|
| 谁触发 | 用户对话 | 主 Agent 在执行任务时调 `task` 工具 |
| 入口 | 左侧栏 Agent 选择器选中后对话 | 装备到主 Agent 后，主 Agent 自主决定何时调 |
| 上下文 | 持续多轮对话 | 每次调用独立隔离，任务完成就结束 |
| 适合做什么 | 长期陪伴的全能助手 | 专家型子任务: 代码审查 / 深度研究 / 数据分析 |

**禁止递归**: 子 Agent 不能再启动孙 Agent，防止递归爆炸。

**装备伙伴**: 在主 Agent 配置页的"伙伴"标签页装备。装上后主 Agent 的 system prompt 自动带"我有 X 伙伴可调"，自主决策何时用。

## Agent 版本 (流动版 vs 受控版)

每个 Agent 有发布策略 (release_policy):

| 模式 | 行为 | 适合 |
|---|---|---|
| **流动版** (auto) | 改动即生效，没有"草稿 vs 正式"区分 | 个人 / 小团队 / 内部使用 / 快速迭代 |
| **受控版** (manual) | 改动是**草稿**，不影响用户。必须点"发布"按钮才生效 | 对外场景 / 严肃产品 / 多人协作 |

**切换入口**: Agent 配置页 → "发布策略" 按钮。

**用户报"我做的修改 Agent 没生效"** — 大概率是受控版下没发布。让用户在 Agent 配置页找"发布"按钮。

**用户报"无权创建 / 无法使用"**: 受控版下普通成员看到的是"上次发布的稳定版"，草稿态资产对他们不可见 — 切流动版或显式发布即可。

## 大模型接入 (LLM 配置)

平台默认带几个内置模型，用户也可以**接入自己买的大模型** (BYOK — Bring Your Own Key)。

**入口**: 设置 → 大模型接入 → "+ 添加模型"。

每个 LLM 配置包含:
- **provider**: openai / anthropic / deepseek / 智谱 / 月之暗面 / 豆包 / Ollama / 自定义 endpoint
- **模型 ID**、**API Key**、(可选) **Base URL** (用代理或自建端点时填)
- **共享方式**:

| 共享方式 | 说明 |
|---|---|
| **团队共享** | 配置一份，团队所有成员都能用。用 owner 的 API Key |
| **个人接入** | 每个成员各自配自己的 Key，互不可见 |

**Agent 不能帮用户接入新模型** — 这是设置页操作。用户问"怎么接 X 模型"指引到设置 → 大模型接入。

## 接入 API (BYOK 第三方工具)

**纠错: 这是把第三方工具的 API Key (例如 Tavily 网页搜索、SearchAPI) 接入平台**，让 Agent 调对应工具时**用你自己的 Key**而不消耗平台默认配额。

**入口**: 设置 → 接入 API。

典型场景: 平台内置的网页搜索工具默认每天有调用次数限制；你接入自己的 Tavily Key 后，搜索调用走你的额度，不再受限。

**不要混淆**:
- **大模型接入** = 给 Agent 接入第三方**大模型**的 Key
- **接入 API** = 给 Agent 接入第三方**工具**的 Key

**外部系统通过 API 调用 Agent**: 后端代码层面有这个能力 (REST + X-API-Key)，但**目前从产品 UI 没有开放给用户**。如果用户问"我能不能让我的系统调用 Agent" → 老实说"目前 UI 没开放给用户配置外部调用 API Key，建议在小精灵反馈面板提一条让团队优先排"，不要瞎指引。

## 连接凭证 (Connections)

外部 SaaS 服务 (飞书 / GitHub / Notion / OpenWeather / 公司自建 API ...) 的认证凭据。**Agent 调用工具或技能时自动从这里读取**，不用每次问用户要 API Key。

**入口**: 设置 → 连接凭证。

关键概念:

- **一个连接器** = 对接一个外部服务。包含**多个字段** (api_key、app_secret、base_url 等)
- 字段有 **owner_level**:
  - **team** 字段: 管理员配一次，全员共享 (典型: app_id / app_secret / 公司 API Key)
  - **user** 字段: 每人各自填，互不可见 (典型: 每个成员自己的 user_token / personal Key)

> "团队中一起用 vs 各用各的"靠**字段的 owner_level** 决定，不是靠"建几个连接器"。一个连接器**同时**有 team 字段和 user 字段是常态。

详细组合形态见 `connections.md`。

## 频道 (Channel) vs CLI 套件

详见 `channels-im.md`。简要区分:

| 维度 | CLI 接入套件 (cli_managed) | 频道接入 (channel binding) |
|---|---|---|
| 是什么 | 把外部 IM 的 API 接到平台，**作为工具能力** | 把外部 IM **作为对话入口** — 用户在 IM 里 @bot 跟 Agent 对话 |
| 怎么用 | Agent 主动调工具发消息 / 创建文档 / 查日历 | 用户在 IM 群里 @bot，像 Web 端一样跟 Agent 对话 |
| 入口 | 设置 → 能力套件 | 设置 → 频道 / Agent 配置页 → 频道 |
| 互不依赖 | 装了 CLI 套件 ≠ 自动有频道 | 绑了频道 ≠ 自动能调 IM 工具 |

## 团队背景 (Team Context)

让 Agent 在所有任务中自带的"团队整体信息" — 例如"我们公司是做 BI 分析的，客户主要是金融行业，产品叫 X"。Agent 的 system prompt 自动带上这段，不用每次解释。

**入口**: 设置 → 团队背景。

类似 OpenAI 的"自定义指令" / Cursor 的 `.cursorrules`，但是**空间级**，对所有 Agent 都生效。

## 文件: 共享文件 vs 会话文件

| 类型 | 在哪里 | 谁能看 | 适合什么 |
|---|---|---|---|
| **会话文件** | 当前会话目录 | 同会话的人 | 临时的、本次任务的产出 |
| **共享文件** | Agent 的 files 目录 (跨会话) | 同 Agent 的所有会话 | 长期复用的资料 / 模板 / 知识库 |

**入口**: 左侧栏 → 共享文件 / 会话文件 (各自一个分区)。

**Agent 在脚本里**: read / write 工具默认指向**会话目录**。想读共享文件，用 `files/xxx` 前缀。

---

## 分享: 分享文件 vs 分享会话

两种分享，都生成公开链接，不用登录就能看，**都是永久有效**的:

| 类型 | 分享什么 | 入口 |
|---|---|---|
| **分享文件** | 单个文件 (HTML / PDF / Markdown / 图片 / Excel ...) | 文件预览页右上的"分享"按钮、或工作室面板里的分享按钮 |
| **分享会话** | 一整个对话 (回放 + 工作室产出文件) | 聊天页面右上角的分享按钮 |

> 平台的分享**没有过期时间选项**，全部是永久链接。不要凭"行业经验"瞎说"30 天 / 7 天有效" — 错的。

**会话分享**支持:
- **回放**: 访客看到的是录像式回放 (消息一条条出现)，适合给同事看你跟 Agent 怎么解决问题
- **是否允许查看上传文件**: 默认勾选 (用户上传给 Agent 的文件能看)，取消则只看到对话
- **工作室面板**: 访客也能打开会话期间 Agent 产出的文件

## 平台助手 (小精灵)

左下角的**小精灵图标** = "平台助手"。不是某个具体 Agent，是平台层面的**反馈与监管入口**。

用户可以:
- **点击小精灵图标** 主动表达不满 / 提建议
- **系统自动弹反馈面板**: 检测到用户表达困惑 / 抱怨时主动邀请反馈
- 在设置中给它换名字、关掉自动感知

**Agent 不要主动调"反馈"** — 让用户自然触发或自己点。详见 `feedback.md`。

## 探索市场

公开的资产市场 — 团队 / 个人发布的 Agent / 技能 / 工具 / MCP / 卡片 / 能力套件都可以发布到这里，其它团队浏览、复制、安装。

**入口**: 左侧栏 → 探索 / 设置 → 智能体资产 → 探索市场。

资产管理 (找 / 装 / 装备 / 删除 / 发布) 详见 `assets-management.md`。
