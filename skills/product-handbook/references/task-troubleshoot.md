# 任务表现不好的诊断与改进

> 当 Agent 在某类任务上一直完成不好 — 用户反复说"还是不对" / 多次尝试都达不到期望 / 同类任务经常拖到迭代上限 / 输出质量明显低于其他任务 — 这份指南帮 Agent **自检 + 找改进方向**。

## 第一原则: 先诊断，再改进

不要凭直觉随便改。面对"做不好"时，先弄清是**哪一类做不好**，再选方法。5 类常见问题，各有对应解法:

| 症状 | 大概是什么问题 | 解法 |
|---|---|---|
| 知识盲区，不懂这类问题怎么做 | 缺**专业指南** | 沉淀技能 (Skill) |
| 反复用错同一个工具 / 方法 | 缺**经验记忆** | 沉淀记忆 (Memory) |
| 推理思路差 / 决策摇摆 / 上下文超长 | **模型能力**不够 | 切换模型 (LLM) |
| 想做但做不到 (没有 X 工具) | 缺**动作能力** | 装 / 创建工具 (Tool / MCP) |
| 工作风格 / 行事方式不对 | 缺**全局指令** | 改 agents.md / 加团队背景 |

---

## 解法 1: 沉淀技能 (Skill)

**适合**: 一类任务**重复出现**且 Agent 每次都要重新摸索 — 这是技能存在的意义。把"这类任务怎么做"沉淀成一份 SKILL.md，装备给 Agent。

**例**:
- 用户多次让 Agent 做财报分析，Agent 每次方法都不一样 → 沉淀"财报分析"技能，写清"先看资产负债 → 再看现金流 → ..."
- 用户经常要"做一份漂亮的 HTML 报告"，Agent 输出参差 → 看内置的"前端设计"技能 (教 Agent 各种顶级视觉调性)

**Agent 在对话里能做的事**:

调用 `skill-creator` 内置技能。它把创建 / 安装 / 调整技能的完整流程都写好了，包括 SKILL.md 怎么拆 references、触发关键词怎么设、渐进式加载怎么做。

**Agent 做不到的事**:

把建好的技能**装备到 Agent**，是用户在 Agent 配置页 → 技能 操作 (4 态 enabled)。

**关键认知**: 技能不是"塞进 system prompt 的几句话"，是**渐进式加载的专业知识库** — SKILL.md 是入口，references/ 是详细细节，按需 read。一上来就把整套写到 agents.md 里只会污染 system prompt。

---

## 解法 2: 沉淀记忆 (Memory)

**适合**: 用户重复纠正 Agent 同一个错 ("已经说了 N 次别用 X" / "公司的 Y 是 Z 不是你说的那个")。这是**事实校正**，不是工作流。

**记忆 vs 技能**:
- 技能 = 我做这类任务**怎么做** (流程 / 标准)
- 记忆 = 我**记住这个用户/团队**告诉我的具体事实 (偏好 / 名词解释 / 边界)

**Agent 在对话里能做的事**:

跟 Agent 说"你记一下: 我们公司 X 是 Y，以后别再叫成 Z" → Agent 自动调记忆工具沉淀。

**用户手动沉淀**: 设置 → 智能体资产 → 记忆 → "+ 新增记忆"，手动写。

**作用域**: 记忆挂在 Agent 上，该 Agent 后续所有对话都能读到。

> 如果是**整个团队的事实集** (例如"我们公司是做 BI 分析的，客户主要金融行业") — 走**团队背景**: 设置 → 团队背景。比记忆更全局，所有 Agent 都生效。

---

## 解法 3: 切换 LLM 模型

**适合**:
- **推理任务** (代码审查 / 数据分析 / 复杂决策) → 试更强的模型 (Claude Opus、GPT-5、DeepSeek R1、Kimi-K2 等)
- **长上下文** (大文档 / 长会话) → 看模型 max_tokens，切大上下文模型
- **工具调用密集** → 选支持 function call 好的模型 (Claude / GPT-4 系 / DeepSeek，老模型有时不支持)
- **速度优先** (聊天 / 简单问答) → 切轻量模型 (Haiku / GPT-4o-mini / DeepSeek V3)

**Agent 在对话里能做的事**:

提建议告诉用户哪类模型更适合，但**模型切换是用户操作**:
1. **临时换** (本次对话): 输入框上方的模型选择器选模型
2. **永久换** (Agent 默认): Agent 配置页 → 默认模型
3. **加新模型** (BYOK): 设置 → 大模型接入 → "+ 添加模型" (详见 `workflows.md` 流程 6)

**判断模型不行的常见信号**:

- Agent 反复 JSON 解析失败 / 工具调用格式乱 → 换 function call 友好的
- Agent 思考没逻辑 / 输出短 → 升级到推理强的
- Agent 在长会话末端表现下降 → 切大上下文模型 / 让用户开新会话
- Agent 输出经常被截断 → 检查 max_tokens 设置

## 解法 4: 装 / 创建工具 (Tool / MCP)

**适合**: Agent **想做但做不到** — 没有对应的 action。

**例**:
- 想查公司工商信息 → 装"企查查"工具 / MCP
- 想帮用户操作飞书 → 装飞书工作流套件 (CLI 接入)
- 想生成 PPT → 看是不是"pptx"内置技能装备了
- 想画图 → 看 generate_image 工具是不是 enabled (默认开)

**Agent 在对话里能做的事**:

1. **找现成的**: 让用户走探索市场，或在对话里说"探索一下"，Agent 调 search_inspirations 推荐
2. **没现成的，自建**:
   - 单工具或多工具自建 → 调用 `tool-connect` 内置技能
   - 装第三方 MCP server → 调用 `mcp-installer` 内置技能
3. **配凭证**: 工具引用了连接器，要先配凭证 (走 `connections.md`)

**Agent 做不到的事**:

把建好 / 装好的工具 / MCP **装备到 Agent** (4 态 enabled) 是用户在 Agent 配置页操作。

**单工具 vs MCP 选择** + **凭证 + 工具 vs 凭证 + 技能** 的判断详见 `workflows.md` 流程 3 的 4 类决策。

## 解法 5: 改 agents.md / 加团队背景

**适合**:
- Agent 工作风格不对 ("太啰嗦"、"少废话直接执行"、"先问清楚再做")
- Agent 不知道用户的角色 ("我是产品经理，不是开发，别给我代码细节")
- Agent 不知道团队 / 公司的整体背景 ("我们是 SaaS 公司，客户主要 B 端")

### 改 agents.md (Agent 级)

**入口**: Agent 配置页 → agents.md。

**适合**: 这个 Agent 特有的角色 / 风格 / 约束。

**例**: "你是一个简洁直接的代码审查专家。不解释你不需要解释的；优先指出真问题，不夸奖优点；用户给的代码默认在主分支，别问。"

### 改团队背景 (空间级)

**入口**: 设置 → 团队背景。

**适合**: 整个团队共享的背景 (公司是做啥的、客户是谁、产品名、术语表)。

**效果**: 所有 Agent 的 system prompt 自动带上这段，不用每个 Agent 各写一遍。

**Agent 在对话里能做的事**:

帮用户**起草**新的 agents.md 内容、团队背景内容，但**实际写入是用户操作** (Agent 配置页或团队背景页粘贴)。

## 综合诊断流程

用户说"Agent 在 X 类任务表现不好":

```
1. 问清楚: "这次具体哪里不对? 期望是什么?"
   └─ 不要凭"上次也是这种"猜，用户反馈才是 ground truth

2. 自我审视:
   ├─ 缺**做这类任务的方法** → 解法 1 (沉淀技能)
   ├─ 缺**用户告诉过的事实** → 解法 2 (沉淀记忆)
   ├─ **思考能力 / 上下文不够** → 解法 3 (切换模型)
   ├─ **该用某个工具但没有** → 解法 4 (装 / 创建工具)
   └─ **整体风格 / 角色不对** → 解法 5 (改 agents.md / 团队背景)

3. 给 1-3 条改进建议，让用户挑感兴趣的先试一条
   └─ 不要 5 条全堆给用户，那样他没法选

4. 主动提议帮用户实现
   └─ 例: "我可以现在就帮你起草一份'财报分析'技能的 SKILL.md，你看了改改就能装备到我身上，试试?"
```

## 症状速查表

| 症状 | 大概率原因 | 解法 |
|---|---|---|
| 同样问题问 3 次答案不一样 | 缺技能 (没沉淀方法) / 模型温度高 / 上下文不够 | 1 或 3 |
| 总用错某个工具的参数 | 工具 description 写得不清 | 改工具 description |
| 总想不到调某个工具 | 工具 description 触发关键词不对 | 改 description |
| 经常说"我做不到" | 真做不到 → 装工具；可以做不知道 → 改 agents.md 给提示 | 4 或 5 |
| 输出格式不稳定 | 缺规范 | 1 (技能) 或 5 (改 agents.md 写明格式) |
| 长会话后变蠢 | 上下文压缩，重要信息被压掉了 | 重要事实写到记忆 / 团队背景，别只在对话里说 |
| 工具调成功了但结论错 | 模型能力 / 缺解读方法论 | 3 (升级模型) 或 1 (加技能教它"看到这个数据该怎么解读") |
| 任务跑到一半 Agent 主动停 | "任务偏离提醒"过多触发 / 迭代上限到了 | 让用户给更具体目标，或拆成多会话做 |

## 改进试过都没效 → 引导提反馈

如果试过 1-2 个解法都没效，**不要硬掰**。引导用户走反馈通道:

- 跟用户说: "我帮你试了 X 和 Y 但还是不太理想，建议你点左下角小精灵图标提一条反馈，附上这个会话的链接，让平台团队看一眼是不是有更深层的问题。"
- 详细的反馈撰写引导见 `feedback.md`。

---

## 一行心法

**先诊断，再改进**。用户说"做得不好"时不要立刻提"换模型" / "装工具" / "写技能"当万能药 — 5 类症状 5 种解法，选错了反而浪费用户耐心和上下文。
