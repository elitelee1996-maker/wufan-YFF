# 智能体资产 · 找 / 装 / 装备

> 用户问"这个能力 Agent 怎么获得?" / "我想找一个 X 工具" / "怎么安装 / 装备 / 切换" — 三步: **找 → 装 → 装备**。

## 资产类型 5 + 1

5 种资产 (详见 `concepts.md`):

| 类型 | 是什么 | 怎么用 |
|---|---|---|
| **技能** (Skill) | SKILL.md + 配套资源 | 装备到 Agent，触发关键词时 Agent 自动 read |
| **工具** (Tool) | 一段可执行 Python | 装备到 Agent，Agent 直接调用 |
| **伙伴** (子 Agent) | 独立的 Agent 实例 | 装备到主 Agent，主 Agent 通过 task 调 |
| **MCP** | 一组工具 (协议封装) | 装备到 Agent，Agent 调里面的工具 |
| **卡片** (Widget) | UI 组件 | 装备到 Agent，渲染给用户做交互 |

加 + 1: **能力套件** (Capability Pack) — 资产组合包，一次装一组。

---

## Step 1: 找资产

### 路径 A: 探索市场

**入口**: 左侧栏 → 探索 / 设置 → 智能体资产 → 探索市场。

里面有平台 / 其它团队公开发布的资产，可以浏览、搜索、按类型 / 场景 / 热度筛选。

适合:
- 不知道有什么现成的 → 浏览市场看看
- 想要某类能力 → 搜关键词 (例: "飞书"、"代码审查"、"数据分析")

### 路径 B: 对话里直接探索

跟 Agent 说"探索一下" / "我想找些灵感"，Agent 会调内置 `search_inspirations` 工具，给个性化推荐资产 / 能力包 / 内容范例。

不止是搜资产，还会推荐**相关的玩法 / 案例**。适合"还在选题阶段、不确定自己想要什么"时。

### 路径 C: 从外部生态找

用户可能从 GitHub / 社区拿到一份现成的 SKILL.md / 工具代码 / mcp.json，通过下面的"装"步骤导入。

---

## Step 2: 装资产

### 内置 / 平台已有的资产

在探索市场看到喜欢的，点"安装" / "添加到我的空间"，直接进当前空间的资产池，之后可装备到任意 Agent。

### 来自 GitHub / ZIP / 外部链接

#### Agent 在对话里能做的事

跟 Agent 说"帮我装这个: [GitHub 链接 / ZIP 链接]" — Agent 根据资产类型，调用对应内置技能去装:

| 装什么 | Agent 调用哪个内置技能 |
|---|---|
| 技能 (SKILL.md + references) | `skill-creator` (技能创建) |
| 工具 (tool.json + 可选 main.py) + 配凭证 | `tool-connect` (工具与连接) |
| MCP server | `mcp-installer` (MCP 服务安装) |
| 伙伴 (agent.md) | `subagent-creator` (伙伴创建) |

支持的来源:
- GitHub 仓库 URL (含 SKILL.md / tool.json / mcp.json 的目录)
- 单文件 raw URL
- ZIP / .skill 包

#### Agent 做不到，必须用户手动操作

入口: 设置 → 智能体资产 → 选对应 tab (技能 / 工具 / MCP / 卡片) → "+ 新建" / "+ 上传"

填:
- **技能**: 上传 SKILL.md + 关联文件
- **工具**: 写一段 Python `execute(ctx, ...)` 函数
- **MCP**: 填 server URL / 命令 + 凭证依赖
- **卡片**: 上传 React 组件代码

适合用户自己开发、不在 GitHub 公开的内部资产。

### 安装能力套件

普通能力套件直接"安装"即可。

**CLI 套件特殊** (cli_managed) — 飞书 / 钉钉 / 企微，安装时要走扫码授权，详见 `channels-im.md`。

---

## Step 3: 装备到 Agent

资产装到空间 ≠ Agent 能用。还要**装备**给具体 Agent。

**入口**: Agent 配置页 / 设置 → 智能体资产 → 选 Agent → 装备清单。

### 4 态绑定

| 态 | 含义 |
|---|---|
| **enabled** (启用) | Agent 一直能用 |
| **disabled** (禁用) | 装着但 Agent 看不到，用不了 |
| **optional** (按需) | Agent 默认看不到，用户在对话能力选择器里勾上才用 |
| **locked** (锁定) | 强制装上，用户不能取消 (典型: 平台默认带的核心工具) |

> 默认新装的资产是 **enabled**。不想要某个资产但又不想删 → 切到 disabled。

### Agent 不能装 / 卸装备

装备、卸装备、改 4 态都是用户在 Agent 配置页操作。Agent 自己只能"建议要不要装"、"建议什么资产装上"，不能直接动用户的装备清单。

---

## 流动版 vs 受控版对资产的影响

| 模式 | 行为 |
|---|---|
| **流动版** (auto) | 装一个新资产，Agent 立刻能用 |
| **受控版** (manual) | 装新资产 → 进 draft → 普通成员看不到 → 必须点"发布"才进 production |

切换入口: Agent 配置页 → "发布策略"。

> Agent 不能切换发布策略。是用户 / 管理员操作。

---

## 删除 / 卸载资产

**入口**: 设置 → 智能体资产 → 找到资产 → 删除。

区分:
- **删除整个资产**: 从空间彻底卸载，所有 Agent 都不能用了
- **仅取消装备**: 资产留在空间里，但当前 Agent 不用 (其它 Agent 还能用)

---

## 发布资产到探索市场

用户做了一个不错的资产想分享出去:

**入口**: 设置 → 智能体资产 → 找到资产 → "发布到市场" / 探索 → 我的发布 → "+ 发布资产"。

发布向导:
1. 选要发布的资产
2. 填介绍 / 截图 / 标签
3. 选可见性 (公开 / 仅特定团队)
4. 提交

平台会做 AI 预审 (合规检查、敏感信息检查)，通过后上架。

**发布后不会自动同步更新** — 资产改了之后，想让市场用户用上新版，要在"我的发布"里点"更新"。

---

## 做不好怎么办: "我装上了为啥用不了"

按以下顺序排查:

1. **是不是装备到当前 Agent 了?** 左侧栏当前选中的 Agent 才是装备生效的对象，装备页选错 Agent 都白搭
2. **绑定状态是 enabled 吗?** 4 态里 disabled / optional 没勾，Agent 看不到
3. **流动版 vs 受控版?** 受控版下普通成员只能用已发布版本，草稿态资产对他们不可见 → 切流动版或显式发布
4. **凭证 / 依赖配齐了吗?** 工具 / MCP 经常需要凭证，没配齐就报错 (走 `connections.md` 排查)
5. **LLM 模型支持工具调用吗?** 老模型 / 自建小模型可能不支持 function call，装了也用不上

排查完没结论 → 让用户在小精灵反馈面板提一条，附上 Agent ID、资产名、报错信息。

## 用户问"什么时候用技能? 什么时候用工具? 什么时候用 MCP?"

| 想给 Agent | 装什么 |
|---|---|
| 一个**新动作能力** (调 API / 跑代码 / 操作文件) | 工具 |
| 一份**专业指南 / 工作流** (XX 类任务该怎么做) | 技能 |
| 一组**第三方协议工具集** | MCP |
| 一个**专业子任务执行者** (要独立上下文) | 伙伴 |
| 一种**给用户的 UI 交互** | 卡片 |

经常**多种组合**: 例如做"财报分析" — 装"财报分析"技能 (告诉 Agent 怎么做) + 装"PDF 解析"工具 (让 Agent 能读 PDF) + 装"Excel 输出"工具 (让 Agent 能生成 Excel)。组合判断详见 `workflows.md` 流程 3。

---

## 一行心法

资产是**装到空间、装备到 Agent、由用户操作**的 — Agent 不能装 / 卸 / 装备 / 卸装。用户问"我装上了为啥用不了"先**排查 4 态绑定 + 流动/受控版 + 凭证**，不要瞎归因到"模型不行"。
