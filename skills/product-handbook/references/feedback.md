# 反馈通道指引

> 用户说"我有个建议" / "这功能不好用我想反馈" / "怎么提个 bug" — 给到**最匹配的反馈通道**，别让用户白跑。

## 平台的反馈入口

| 入口 | 在哪 | 适合什么 |
|---|---|---|
| **小精灵主动弹的反馈面板** | 系统自动 (检测到用户表达不满 / 困惑 / 抱怨) | 当下不爽的具体场景 |
| **点击小精灵图标** | 左下角小精灵 | 主动想反馈但没想好类型 |
| **@ 小精灵** | 输入框 @ 小精灵 (打 @ 触发命令补全) | 在对话里**就近反馈** ("这次回答完全不对") |
| **消息级 👍👎** | Agent 单条消息工具栏 | 评价单条回复，👎 会自动弹反馈面板 |

不同入口主要差**触发时机**和**上下文记录的精确度**，最终都进同一个反馈池，后台有平台运营 / 产品看。

---

## 怎么引导用户

### 场景 1: 用户说"我想提个 bug"

> "你直接点左下角小精灵图标，弹出来的反馈面板里选 'Bug 反馈'，描述清楚:
> - 当时在做什么 (操作)
> - 期望是什么 (你以为应该的样子)
> - 实际发生了什么 (问题现象)
> - 如果能复现，写下复现步骤
>
> 后台收到会有人跟进。"

### 场景 2: 用户说"我有个产品建议"

> "可以点小精灵图标，选 '功能建议'，把想法写进去。写得越具体越好 — 不是只说'希望支持 X'，而是 '我做 Y 任务时遇到 Z，如果支持 X 就能解决'。这样平台运营更容易判断优先级。"

### 场景 3: 用户在对话里直接吐槽 ("这回答完全不对")

平台**自动**会在用户表达不满 / 困惑 / 抱怨时弹小精灵反馈面板邀请反馈 (基于消息内容感知)。Agent 不要主动调反馈通道，让用户自然触发。

但 Agent **应该**:
- 真诚回应不满，不要装没听见
- 主动问"哪里不对? 我重做一下" — 这是修复，比反馈更重要

如果用户**就是想吐槽不想再试**，提示: "你也可以点左下角小精灵，留下来给团队看一眼，帮我改进。"

### 场景 4: 用户对单条回答 👍 / 👎

不用引导 — 用户在 Agent 单条消息上方有 👍👎 工具栏。点 👎 时平台**自动**弹反馈面板让用户写细节。

### 场景 5: 用户想把"建议"提到能给所有用户看的地方

平台没有公开的"社区讨论"入口。让用户走"功能建议"，后台决定是否对外公开 (有些会进 changelog)。

## 反馈类目 (实际 panel 弹出来的选项可能略不同)

| 类目 | 含义 |
|---|---|
| **Bug 反馈** | 功能坏了 / 报错 / 跟说明不符 |
| **事实错误** | Agent 给的答案不准 / 编造 |
| **体验问题** | UI 不顺手 / 操作绕 / 找不到入口 |
| **功能建议** | 想要新功能 / 改现有功能 |
| **性能 / 速度** | 卡 / 慢 / 经常超时 |
| **内容质量** | Agent 输出不达预期 (跟"事实错误"区别: 不是错，是不够好) |
| **其它** | 不属上面分类的 |

## Agent 何时该建议用户提反馈

不是任何时候都让用户提。Agent 自己能解决的，不要推给反馈通道。

| 情形 | 建议提反馈? | 备注 |
|---|---|---|
| Agent 自己能改进 (装工具 / 写技能 / 切模型) | **不要**先推反馈 | 先帮他改进，见 `task-troubleshoot.md` |
| 用户问产品操作，Agent 不知道 | **不要**推反馈 | 先 read 本技能找答案 |
| 平台某个功能确实坏了 (有报错截图) | 推反馈 + 教用户写清楚现象 | bug |
| 用户要的功能确实平台没有 | 推反馈 + 鼓励写**用例** ("我做 Y 时希望 X 解决...") | 建议 |
| 用户表达对 Agent 的不满 | **不主动推**，等他自然抱怨时平台自动弹 | 让平台感知系统接管 |
| 用户表达对平台 / UI / 体验的不满 | 推反馈 | 这不是 Agent 能改的，是产品的事 |
| Agent 试了多种解法都没解决 | 推反馈 | 兜底，附上当前会话和尝试过程 |

> Agent 不应该把"提反馈"当万能甩锅。用户问"怎么 X"，先老实查 (本技能 / 内核技能)，查不到再说不清楚 / 建议提反馈。

## 写好反馈的指导 (用户问"怎么写"时给)

```
1. 题目: 一句话说清核心问题 (50 字内)
   ✗ 不好用                                         ← 太空
   ✓ 飞书 CLI 安装时扫码后提示"等待审核"无法继续  ← 具体

2. 具体场景: 我在做什么? 期望什么? 实际什么?
   ✗ 老是出错                       ← 没说哪
   ✓ 在 Agent 配置页装备"qcc 企查查"工具时，点装备按钮没反应，网络也没请求发出

3. 复现步骤 (bug 必填):
   1. 进入 设置 → 智能体资产 → 工具
   2. 点 Agent A 的"装备"按钮
   3. ...

4. 截图 / 录屏: 越能让人重现问题，修复越快

5. 影响范围: 我自己挂了 / 团队都挂了 / 全平台
   - 这帮平台判断严重程度
```

---

## 一行心法

反馈通道是**用户对产品的最后一公里**。该走反馈的别在对话里硬掰；Agent 能解决的别推反馈。关键是判断: **这事 Agent 能不能搞定 / 是不是 platform 层面的事 / 用户是想抱怨还是想推动改进**。
