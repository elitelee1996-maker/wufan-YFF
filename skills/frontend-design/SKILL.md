# 前端设计

此技能指导创建独特的、生产级的前端界面，避免千篇一律的"AI 流水线"审美。实现真正可用的代码，对美学细节和创意选择给予极致关注。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能会包含关于用途、目标受众或技术约束的背景信息。

## 设计思维

在编码之前，理解背景并确立一个**大胆**的美学方向：
- **目的**：这个界面解决什么问题？谁在使用它？
- **调性**：选择一个极端风格：极简主义、极繁主义混沌、复古未来主义、有机自然风、奢华精致、趣味玩具风、杂志编辑风、粗野主义、装饰艺术/几何风、柔和粉彩、工业实用主义等。有太多风格可供选择。以这些作为灵感，但设计出真正符合所选美学方向的作品。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：是什么让这个设计令人**难忘**？用户会记住的那一点是什么？

**关键**：选择一个清晰的概念方向，并精准地执行。大胆的极繁主义和精致的极简主义都可行——关键在于意图性，而非强度。

然后实现可工作的代码（HTML/CSS/JS、React、Vue 等），要求：
- 生产级且功能完善
- 视觉冲击力强且令人难忘
- 具有清晰美学观点的整体协调性
- 每个细节都经过精心打磨

## 前端美学指南

关注以下方面：
- **字体排版**：选择美观、独特、有趣的字体。避免使用 Arial 和 Inter 等通用字体；转而选择能提升前端美感的独特字体；出人意料、有个性的字体选择。将独特的展示字体与精致的正文字体搭配使用。
- **色彩与主题**：坚持统一的美学风格。使用 CSS 变量保持一致性。主导色配合鲜明的点缀色，胜过平淡、均匀分布的色彩方案。
- **动效**：使用动画实现效果和微交互。HTML 项目优先使用纯 CSS 方案。React 项目可用时使用 Motion 库。专注于高影响力的时刻：一个精心编排的页面加载动画配合交错显示（animation-delay）比零散的微交互更能带来愉悦感。使用滚动触发和令人惊喜的悬停状态。
- **空间构图**：出人意料的布局。不对称。重叠。对角线流动。打破网格的元素。大量留白或有控制的密集感。
- **背景与视觉细节**：创造氛围和深度，而非默认使用纯色背景。添加与整体美学相匹配的上下文效果和纹理。运用创意形式，如渐变网格、噪点纹理、几何图案、层叠透明度、戏剧性阴影、装饰性边框、自定义光标和颗粒叠加。

**绝对不要**使用千篇一律的 AI 生成审美，比如过度使用的字体系列（Inter、Roboto、Arial、系统字体）、老套的配色方案（特别是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏特定场景个性的模板化设计。

创意性地诠释，做出令人意外的选择，让设计真正为特定场景量身定制。每个设计都应该不同。在亮色和暗色主题之间变化，使用不同的字体，不同的美学风格。**绝对不要**在多次生成中趋同于常见选择（例如 Space Grotesk）。

**重要**：实现复杂度应与美学愿景相匹配。极繁主义设计需要精心编排的代码，包含大量动画和效果。极简主义或精致设计需要克制、精准，以及对间距、字体排版和微妙细节的细致关注。优雅来自于对愿景的出色执行。

## 图片素材 · 顶尖设计的隐藏一招

CSS 渐变 / 噪点 / 几何 / 纹理能撑起 80% 的前端美学 — 9 套情绪板里大部分元素都是纯 CSS, 不依赖任何位图. 但**剩下那 20%** — Hero 主图、邀请函封面、海报背景、人物肖像、产品照、品牌插画、装饰摄影 — **顶尖图片素材就是顶尖前端跟"看上去 OK"的最大差距**. 没有它, 再好的字体排版也撑不起"震慑感".

平台自带两个工具, 你可以**主动**用来给作品配顶尖素材, 不用先问用户:

- `generate_image(prompt, filename, aspect_ratio)` — 文生图, 适合: Hero 背景 / 邀请函主图 / 海报主视觉 / 产品概念图 / 装饰插画 / 人像肖像
- `edit_image(images, prompt, filename)` — 改图 / 合成, 适合: 已有 logo 换背景、参考图换调性、几张图合成构图

**判断要不要生图的 3 个问题** (任一为 yes 就考虑生):
1. 这个位置如果空着 / 用纯色 / 用几何, 视觉冲击力是不是会差一档?
2. 用户的内容气质是不是需要一张"传神的图"才表达得到位?
3. 我现在要做的是不是邀请函 / 海报 / Landing 页 hero / 产品故事页 (这些"图主导"的场景)?

**写 prompt 的工艺要求** — 跟你写 CSS 一样讲究:
- 调性必须跟整体视觉语言对齐 (深色赛博风的页就 prompt "cinematic dark neon, ultra-detailed, 8k", 别 prompt "cute cartoon")
- 风格要具象 (摄影风 / 杂志编辑摄影 / 极简插画 / 复古印刷海报 / 3D 渲染 / 油画质感 ...)
- 给细节锚点 (光线 / 材质 / 视角 / 色彩主调), 不要只说 "a beautiful background"

**不必须** — 如果 CSS 能搞定的氛围 (渐变 mesh / 噪点纹理 / 几何线框 / 极光 / 玻璃) 就不要硬塞图. 凡是设计就生图也是另一种廉价. 关键是: 当作品需要"那张顶尖的图"时, 别犹豫, 用 `generate_image` 立刻配上.

## 零 Emoji · 铁律

**绝对不要在产出的 HTML / 文案 / 设计中使用 emoji**, 除非用户明确要求.

emoji 是**廉价设计的最常见元素** — 顶尖前端设计永远靠**字体 / 排版 / 图标 / 色块 / 留白**自身做强调, 不靠 emoji 装饰. 一个 emoji 出现, 整个作品的"专业感 / 高级感 / 设计品质"立刻掉一大档.

**包括但不限于**:
- emoji: 📊 📈 💡 🔍 📌 📦 🚀 🎨 🎯 ✨ ⚡ ⭐ ❗ 🌟 🔥 ✅ ❌ 🎉
- 装饰 unicode (常被 LLM 当 checkmark / 项目符号): ✓ ✗ ★ ☆ ▲ ▼ ● ◆ ■ ◯
- 复合修饰: 「✓ 完成」「📊 数据看板」「💡 洞察」

**用什么代替**:
- 标题强调 → `<em>` 包关键词 (用情绪板的色, 比如玫红 / 钢蓝 / 渐变胶囊) 或 `<strong>`
- 列表分隔 → mono 编号 (`01 / 02 / 03`) 或衬线大数字
- 状态指示 → 文字 ("已完成 · 进行中 · 异常") 或 CSS pill 标签
- 装饰元素 → kicker 横线 / hairline / 几何图形 / 颗粒纹理 / 条纹

**例外可用**: 数学/方向运算符 (← → ↑ ↓ ÷ × ± ≥ ≤) · 排版标点 (· — …).

**自检**: 写完 HTML 后扫一眼有没有这些字符 (Unicode 范围 U+1F300-1F9FF / U+2600-27BF / 还有 ✓ ✗ ★ 等). 有就改掉.
