## 设计理念

**不要制作无聊的幻灯片。** 白色背景上的纯文本列表不会打动任何人。为每张幻灯片考虑以下设计元素。

### 开始之前

- **选择大胆、与内容相关的配色方案**：配色应该让人感觉是为此主题量身定制的。如果把你的颜色换到一个完全不同的演示文稿中仍然"适用"，说明你的选择还不够具体。
- **主导而非均等**：一种颜色应占主导地位（60-70% 视觉权重），搭配 1-2 种辅助色调和一种醒目的强调色。永远不要让所有颜色权重相同。
- **深浅对比**：标题页和结束页使用深色背景，内容页使用浅色（"三明治"结构）。或者全程使用深色调营造高级感。
- **坚持一个视觉母题**：选择一个独特元素并重复使用——圆角图片框、彩色圆圈内的图标、粗边框等。贯穿每张幻灯片。

### 配色方案

选择与主题匹配的颜色——不要默认使用通用蓝色。以下调色板供参考：

| **森林苔藓** | `2C5F2D`（森林绿） | `97BC62`（苔藓绿） | `F5F5F5`（乳白） |

### 每张幻灯片

**每张幻灯片都需要视觉元素**——图片、图表、图标或形状。纯文字幻灯片毫无记忆点。

**布局选项：**
- 双栏（左文右图）
- 图标 + 文字行（彩色圆圈内的图标，粗体标题，下方描述）
- 2x2 或 2x3 网格（一侧图片，另一侧内容块网格）
- 半出血图片（全左或全右侧）配内容叠加

**数据展示：**
- 大数字标注（60-72pt 的大数字配下方小标签）
- 对比列（前后对比、优缺点、并列选项）
- 时间线或流程图（编号步骤、箭头）

**视觉润色：**
- 章节标题旁使用彩色小圆圈内的图标
- 关键数据或标语使用斜体强调文字

### 字体排版

**选择有特色的字体搭配**——不要默认使用 Arial。选择有个性的标题字体，搭配清晰的正文字体。

| 标题字体 | 正文字体 |
|----------|----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

**中文内容字体**（当幻灯片包含中文时使用）：

| 标题字体 | 正文字体 |
|----------|----------|
| WenQuanYi Zen Hei | WenQuanYi Micro Hei |
| WenQuanYi Zen Hei | WenQuanYi Zen Hei |

| 元素 | 尺寸 |
|------|------|
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题 | 20-24pt 粗体 |
| 正文 | 14-16pt |
| 说明文字 | 10-12pt 浅色 |

### 间距

- 最小边距 0.5"
- 内容块间距 0.3-0.5"
- 留出呼吸空间——不要填满每一寸

### 避免（常见错误）

- **不要重复相同布局**——在幻灯片间变换栏、卡片和标注
- **不要居中正文**——段落和列表左对齐；只居中标题
- **不要吝啬尺寸对比**——标题需 36pt+ 才能从 14-16pt 正文中脱颖而出
- **不要默认蓝色**——选择反映具体主题的颜色
- **不要随意混用间距**——选择 0.3" 或 0.5" 间隔并保持一致
- **不要只装饰一张就放弃**——要么全程精心设计，要么全程保持简洁
- **不要做纯文字幻灯片**——添加图片、图标、图表或视觉元素；避免纯标题+列表
- **不要忽略文本框内边距**——对齐线条或形状与文字边缘时，设置 `margin: 0` 或偏移形状以补偿内边距
- **不要使用低对比度元素**——图标和文字都需要与背景形成强对比
- **绝对不要在标题下使用装饰线**——这是 AI 生成幻灯片的标志性特征；用留白或背景色代替

## 质量检查（必须执行）

**假设存在问题。你的任务是找到它们。**

你的第一次渲染几乎不可能完美。把 QA 当作找 bug，而不是确认步骤。如果第一次检查没发现任何问题，说明你看得不够仔细。

### 内容检查

```bash
python -m markitdown output.pptx
```

检查缺失内容、错别字、顺序错误。

**使用模板时，检查残留的占位文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 返回结果，修复后再宣布完成。

### 视觉检查

将幻灯片转为图片（参见[转为图片](#转为图片)），然后**直接使用 `analyze_image` 工具**逐张检查。

**重要规则：**
- **必须自己亲自用 `analyze_image` 检查**——不要委托给 code-reviewer 等子代理，它们没有视觉分析能力
- **每次传一张图片路径**——如 `analyze_image(images="slide-01.jpg", question="...")`
- **在完成所有视觉 QA 之前，不要删除截图文件**

使用以下提示词：

```
视觉检查这张幻灯片。假设存在问题——找到它们。

检查项：
- 元素重叠（文字穿过形状、线条穿过文字、元素堆叠）
- 文字溢出或在边缘/框边界处被截断
- 装饰线为单行文字设计但标题换行成两行
- 来源引用或页脚与上方内容冲突
- 元素间距过小（< 0.3" 间隔）或卡片/区块几乎接触
- 间距不均（一处大面积空白，另一处拥挤）
- 幻灯片边缘边距不足（< 0.5"）
- 列或相似元素未一致对齐
- 低对比度文字（如浅灰文字在奶油色背景上）
- 低对比度图标（如深色图标在深色背景上且无对比圆圈）
- 文本框过窄导致过度换行
- 残留的占位内容

报告所有发现的问题，包括轻微问题。
```

### 验证循环

1. 生成幻灯片 → 转为图片 → 用 `analyze_image` 逐张检查
2. **列出发现的问题**（如果没发现，更严格地再看一遍）
3. 修复问题
4. **重新验证受影响的幻灯片**——一个修复经常引发另一个问题
5. 重复直到完整检查无新问题

**在至少完成一个修复-验证循环之前，不要宣布成功。**

### 文件清理

只有在视觉 QA 全部完成、确认无需修复后，才可以删除中间文件（截图 JPG、PDF、JS 源码）。**最终交付的 .pptx 文件不要删除。**

## 转为图片

将演示文稿转为单张幻灯片图片用于视觉检查：

```bash
soffice --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

生成 `slide-1.jpg`、`slide-2.jpg` 等。

修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

### 字体与中文渲染

soffice 渲染 PDF 时依赖**系统已安装的字体**。如果视觉检查时中文显示为方框（□□□），说明运行环境缺少 CJK 字体——这是**系统级问题**，不是 PPTX 文件或 `fontFace` 设置的问题，不要通过修改 JS 代码中的字体名来修复。

当前环境已预装的中文字体：
- **WenQuanYi Zen Hei**（文泉驿正黑）— 适合标题
- **WenQuanYi Micro Hei**（文泉驿微米黑）— 适合正文

创建包含中文的 PPTX 时，`fontFace` 应使用上述字体名。
