# 网络深度研究

先通过规划模式明确研究方向和范围，再启动多个子代理并行研究，最后综合形成报告。

## 研究流程

### 1. 进入规划模式，收集需求

**首先进入 `plan_mode`**，然后用 `ask_user_question` 收集关键信息，多使用选择题，方便用户回答：

```python
ask_user_question(
  context="为了更好地进行研究，我需要了解一些信息",
  questions=[
    {"id": "scope", "type": "text", "question": "研究的核心问题是什么？希望覆盖哪些方面？"},
    {"id": "depth", "type": "single_choice", "question": "期望的研究深度？", 
     "options": ["快速概览", "中等深度", "深度调研"]},
    {"id": "output", "type": "single_choice", "question": "期望的产出形式？",
     "options": ["口头总结", "简要报告", "详细研究报告"]},
    {"id": "focus", "type": "text", "question": "有特别关注的角度或已知的信息源吗？"}
  ]
)
```

根据用户回答，在 plan 文件中确定：
- 研究问题拆分（独立、不重叠的子主题）
- 每个子主题的研究重点
- **每个子主题的预期产出文件**（中文文件名和路径，产出在 plan 所属文件夹下）
- 预期最终报告结构
- todo清单

**Plan 文件产出规划示例**：

假设 plan 文件路径为 `研究主题/研究主题Plan.md`，则子代理产出应规划在同一文件夹下：

```markdown
## 子主题1：公司概况与发展历程
**研究重点**：
- 公司成立时间、创始团队背景
- 发展历史与关键里程碑
- 融资历程与股权结构

**预期产出**：`研究发现-企业背景.md`

## 子主题2：核心产品体系
**研究重点**：
- 完整产品线介绍
- 各产品功能特点、定位差异
- 产品应用场景与典型客户案例

**预期产出**：`研究发现-产品信息.md`

## 子主题3：技术能力与创新
**研究重点**：
- 核心技术架构与技术特点
- 自研技术与专利情况
- 技术团队实力与研发投入

**预期产出**：`研究发现-技术手段.md`
```

**退出 plan_mode 让用户确认方案**。

### 2. 创建任务清单，开始执行

确认后用 `todo_write` 管理研究进度：

```
1. [in_progress] 启动子代理并行研究各子主题
2. [pending] 读取各子代理发现，初步综合
3. [pending] 撰写最终报告
```

### 3. 并行研究（核心步骤）

使用 `task` 启动多个 网络研究员 子代理。**关键：必须在任务描述中明确指定产出文件的完整路径（plan 文件所在文件夹 + 文件名）**：

```python
# 假设 plan 文件在 "帆软深度研究/帆软深度研究Plan.md"
# 则子代理产出路径为 "帆软深度研究/findings_xxx.md"

task(tasks=[
  {
    "task": """研究帆软软件的公司概况与发展历程。
研究重点：公司成立时间、创始团队、发展历程、融资情况、股权结构。
使用 web_search 搜索相关信息，对官方资料、新闻报道等高价值链接用 web_fetch 精读。
【产出要求】将研究发现保存到 `帆软深度研究/研究发现-企业背景.md`，包含关键事实和来源URL。""",
    "subagent": "网络研究员"
  },
  {
    "task": """研究帆软软件的核心产品体系。
研究重点：FineReport、FineBI 等产品功能、应用场景、客户案例。
使用 web_search 搜索，对产品官网、用户评测等用 web_fetch 精读。
【产出要求】将研究发现保存到 `帆软深度研究/研究发现-产品信息.md`，包含关键事实和来源URL。""",
    "subagent": "网络研究员"
  },
  {
    "task": """研究帆软软件的技术能力与创新。
研究重点：技术架构、自研技术、专利情况、研发团队。
【产出要求】将研究发现保存到 `帆软深度研究/研究发现-技术手段.md`，包含关键事实和来源URL。""",
    "subagent": "网络研究员"
  }
])
```

**子代理研究策略**：
- `web_search`：广度搜索，获取结果列表
- `web_fetch`：深度精读（官方文档、报告等高价值链接必读）
- 每个子主题：3-5 次搜索 + 2-3 次精读
- **产出文件路径必须与 plan 中规划的一致**

### 4. 综合与报告

1. **从 plan 所在文件夹读取各子代理的发现文件**（如 `帆软深度研究/研究发现_*.md`）
2. 综合形成完整答案，引用来源
3. **使用 charts 语法丰富报告表现**（见下方"报告可视化规范"）
4. **最终报告也保存在 plan 所在文件夹**（如 `帆软深度研究/帆软深度研究报告.md`）

## 子代理配置

网络研究员 子代理可用工具：
- `web_search` / `web_fetch`：网络搜索和抓取
- `read` / `write`：文件读写

## 报告可视化规范

**为使报告更专业、更易读，应大量使用 charts 语法将数据和结构化信息可视化。**

### 使用原则

1. **数据必可视化**：有数字对比、占比、趋势的内容，优先用图表呈现
2. **结构必结构化**：多项对比、流程步骤、层级关系，用对应图表类型
3. **忠于事实**：所有图表数据必须来源于研究发现，不可编造数据
4. **适度使用**：每个主要章节建议 1-3 个图表，避免堆砌

### 可用图表类型

在 Markdown 中使用 ` ```charts ` 代码块，内容为 JSON 格式：

#### 数据展示类

**table** - 多维数据对比（最常用）
```charts
{
  "type": "table",
  "title": "竞品功能对比",
  "columns": ["产品", "价格", "核心优势", "市场份额"],
  "data": [
    ["产品A", "¥99/月", "易用性强", "35%"],
    ["产品B", "¥199/月", "功能全面", "28%"]
  ]
}
```

**bar** - 数值对比
```charts
{
  "type": "bar",
  "title": "市场份额对比",
  "data": [
    {"category": "产品A", "value": 35},
    {"category": "产品B", "value": 28}
  ]
}
```

**pie** - 占比分布
```charts
{
  "type": "pie",
  "title": "市场份额分布",
  "data": [
    {"category": "领导者", "value": 45},
    {"category": "挑战者", "value": 30},
    {"category": "其他", "value": 25}
  ]
}
```

**metrics** - 关键指标卡片
```charts
{
  "type": "metrics",
  "title": "核心指标",
  "data": [
    {"label": "营收", "value": "15.7亿", "change": "+12%", "trend": "up"},
    {"label": "客户数", "value": "36,000+"}
  ]
}
```

**line** - 趋势变化
```charts
{
  "type": "line",
  "title": "营收增长趋势",
  "xAxis": ["2021", "2022", "2023", "2024"],
  "series": [{"name": "营收(亿)", "data": [8.4, 11.4, 13.6, 15.7]}]
}
```

#### 结构展示类

**comparison** - 两方/多方对比
```charts
{
  "type": "comparison",
  "title": "方案对比",
  "items": [
    {"dimension": "性能", "方案A": "高", "方案B": "中"},
    {"dimension": "成本", "方案A": "贵", "方案B": "便宜"}
  ]
}
```

**timeline** - 发展历程/时间线
```charts
{
  "type": "timeline",
  "title": "发展历程",
  "data": [
    {"period": "2020", "events": "产品发布", "revenue": "1亿"},
    {"period": "2024", "events": "市场第一", "revenue": "15亿"}
  ]
}
```

**mindmap** - 层级结构/要点展开
```charts
{
  "type": "mindmap",
  "title": "核心竞争力",
  "root": "竞争优势",
  "children": [
    {"name": "技术优势", "children": ["自研引擎", "专利技术"]},
    {"name": "市场优势", "children": ["客户规模", "品牌影响力"]}
  ]
}
```

**flowchart** - 流程/决策树
```charts
{
  "type": "flowchart",
  "title": "选型决策",
  "nodes": [
    {"id": "start", "label": "评估需求", "children": ["opt1", "opt2"]},
    {"id": "opt1", "label": "方案A", "description": "适合大企业"},
    {"id": "opt2", "label": "方案B", "description": "适合中小企业"}
  ]
}
```

**radar** - 多维能力评估
```charts
{
  "type": "radar",
  "title": "能力评估",
  "dimensions": ["技术", "产品", "服务", "价格"],
  "series": [
    {"name": "产品A", "data": [9, 8, 7, 6]},
    {"name": "产品B", "data": [7, 9, 8, 8]}
  ]
}
```

#### 总结类

**summary** - 核心洞察
```charts
{
  "type": "summary",
  "title": "核心结论",
  "insights": [
    {"title": "市场地位稳固", "description": "连续5年保持领先"},
    {"title": "技术壁垒高", "description": "自研核心引擎"}
  ]
}
```

**rating** - 评分评级
```charts
{
  "type": "rating",
  "title": "综合评级",
  "ratings": [
    {"dimension": "推荐指数", "score": 4, "max": 5},
    {"dimension": "风险等级", "level": "低", "note": "财务健康"}
  ]
}
```

### 报告结构建议

一份专业的研究报告应包含：

1. **执行摘要** - 使用 `summary` 或 `metrics` 展示核心发现
2. **背景介绍** - 使用 `timeline` 展示发展历程
3. **数据分析** - 使用 `table`、`bar`、`pie` 展示数据对比
4. **深度对比** - 使用 `comparison`、`radar` 展示多维对比
5. **结论建议** - 使用 `mindmap`、`flowchart` 展示决策框架
6. **风险评估** - 使用 `rating` 展示风险等级

## 注意事项

- `web_search` 只返回摘要，重要链接务必 `web_fetch` 精读
- 最多 3-5 个子代理并行
- **所有产出（子代理发现、最终报告）统一保存在 plan 所在文件夹**
- **任务委派时必须在 task 描述中明确写出产出文件的完整路径**
- **图表数据必须基于研究事实，严禁编造数据**
